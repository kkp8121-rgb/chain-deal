# Fun QA → 개발 세션 핸드오프: 안테1 진입장벽 보완

> QA 세션이 진단·제안까지. **실제 코드 수정·동기화·커밋은 개발 세션이 수행.**

## 문제 (QA 확증)
- 봇 캐주얼 페르소나 안테1 **91% 사망** + 사람 로그 신규 단체유입이 **첫 블라인드에서 증발**(`human-comparison-2026-06-30.md`).
- 리텐션 **1순위** 레버.

## 목표
신규 안테1 통과율 ↑ · 숙련자(마스터리) 영향 최소 · **초반 낙승(리포트 문제2) 악화 회피**.

## 방법 옵션 (개발 세션이 택1 후 적용)

### 옵션 A — 안테1 목표 완화
- `blindBase`(안테1만) 또는 `blindTarget` 안테1 계수 ↓.
- 파일: `index.html`(`blindBase`/`blindTarget`) → 동기화 `run-sim.cjs`(`blindBase`/`blindTarget`) + `balance-check.cjs`.
- trade-off: 신규 ↑ 이지만 **초반 낙승 ↑**(문제2와 상충), 숙련자도 쉬워짐. **비추천**.

### 옵션 B — 안테1 보스 순화 ★추천
- `peasant`(보릿고개: A·2·3 점수 0)를 안테1 풀에서 제외 → `minAnte:2`. 안테1 보스 = 단색저주·무딘칼날(순한 2종)만.
- 파일: `run-sim.cjs`(`BOSSES`/`pickBoss`) + `index.html`(`BOSSES`/`pickBoss`) + `balance-check.cjs` 동기화.
- 근거: 기존 합의(인색한손 등 `minAnte:2` 차단)의 **연장선**. 보릿고개는 저랭크 시작덱(A~8)에 특히 가혹.
- trade-off: 신규 첫 보스만 완화, **숙련자 무영향**(안테1만), **낙승 영향 최소**. 가장 표적적.

### 옵션 C — 신규 온보딩 자원
- 첫 런(`cd_stats.plays===0`)에 시작 손패 +1 또는 리롤 +1.
- 파일: `index.html`(`newGame`/`startBlind`/`getStats`).
- trade-off: 신규만, 숙련자 무영향. 단 **신규 판별 로직 + 카드 불변식**(CLAUDE.md) 주의. 복잡도 ↑.

## 추천: 옵션 B (표적적 · 기존 합의 연장 · 숙련자·낙승 영향 최소)

## 검증 프로토콜 (개발 세션이 적용 후 실행)
1. **보완 전 기준선**: `node tools/funqa/regress.cjs --save` (현재 main = 보완 전 상태).
2. **보완 적용** (옵션 B 코드 수정 + 3파일 동기화).
3. **회귀 측정**: `node tools/funqa/regress.cjs` — 캐주얼 재미 Δ·클리어 Δ ↑ 기대, 마스터리 Δ≈0 확인.
4. **밸런스 가드**: `node tools/run-sim.cjs` 조건부 통과율 + `node tools/funqa/funqa.test.cjs` 드리프트 가드.
5. (배포 후) 사람 로그 재대조 — 신규 첫 블라인드 통과율 ↑.

## ⚠️ 깨지 말 설계 결정 (HANDOVER/메모리)
- **"RNG 순수 랜덤 = 본질"** — 운빨 즉사를 RNG 보정으로 풀지 말 것.
- **안테1 보스 풀 합의**(플레이테스트로 순한 2종) — 옵션 B는 그 연장선이라 안전.
- **초반 낙승(문제2) ↔ 진입장벽 완화(이 보완) 상충** — 옵션 B가 보스만 건드려 낙승 영향이 가장 작음(추천 이유).
- **3파일 드리프트 동기화 필수** + 변경 후 `balance-check`/`funqa.test` 재실행.
