# CHAIN DEAL (체인딜) — Game Design Document **v2 (개정판)**

작성일: 2026-06-17 (v2: 적대적 검증 4렌즈 전면 반영)
장르: 싱글플레이 덱빌딩 로그라이크 / 스코어 어택
플랫폼: 웹 (TypeScript + PixiJS/canvas, 데스크톱·모바일 브라우저)
포맷: 시드런(seed run) 1회 8~15분, 무한 리플레이

> **개정 요지 (한눈에)**: 4개 렌즈가 공통으로 지목한 5대 치명타 — ① 죽은 탭 50%, ② 정적 재정산(가짜 캐스케이드), ③ ±1 사다리 함정, ④ "한 문장 룰" 거짓말, ⑤ 인-라운드 주체성 ↔ 10초 학습성 모순 — 을 **메커닉 차원에서** 수정했다. 핵심 4개 변경:
> 1. **뽑기 → 고르기**: 매 턴 덱에서 1장이 아니라 **3장 중 1장을 고른다**(드래프트). 죽은 탭을 구조적으로 제거(실측: 무연결 확률 66.8% → 29.9%), 발라트로식 "핸드 선택" 주체성을 탭 한 번 안에 회복.
> 2. **통함 규칙 2축·전부 "같으면"**: 색·숫자 둘 다 **"같으면 통함"**. ±1 사다리를 코어에서 제거(부적 옵트인으로 격리). 아이콘은 통함 채널에서 빼고 부적/각인 레이어로 이동. → 진짜 한 문장 룰 + 통함률 33%(통함=특별 사건).
> 3. **체인 = 진짜 캐스케이드**: 통한 카드가 **속성을 전염(승급)**시켜 *다음* 카드가 새로 통하게 만든다. "이미 보이던 줄 다시 세기"가 아니라 판을 실제로 바꾸는 NEW 이벤트.
> 4. **정수 고정소수점 + 구조적 발산 방지**: 모든 배율을 `int×1000`로 저장(부동소수 결정론 깨짐 차단), 곱셈은 정산 1회만(배율은 덧셈으로 누적) → 발산을 캡이 아니라 모델로 차단.

## 1. 도파민 루프 (가장 먼저·가장 자세히)

> 설계 원칙: **모든 행동은 즉각적·과장된 시청각 보상을 주고, 그 보상은 줄을 채울수록 escalation 한다.** 그리고 **"죽은 탭"을 구조적으로 없앴다** — 매 턴 3장 중 통하는 카드를 고를 수 있으므로 무보상 탭은 의도적 선택일 때만 발생한다.

### 1.1 죽은 탭 제거 — 가장 먼저 고친 것 (실측 근거)

검증 렌즈 4개가 모두 지적한 1순위 문제: 원안의 무작위 뽑기는 **통함률 43%, 무연결 탭 67%**. 슬롯머신은 매 스핀이 "항상 무언가" 정산되는데 원안은 절반 이상이 무음 '톡'이었다. 이건 "도파민 없음"이라던 STRANDLOOM 폐기 사유의 직접 재발이다.

**해결: 뽑기 → 고르기 (드래프트 3장).**

```
매 턴: 덱 상단 3장이 손패로 펼쳐진다 (앞면 공개)
       → 플레이어가 1장을 골라 줄 끝에 이어 붙인다
       → 통하는 카드를 고르면 즉시 폭발 / 끊는 카드를 고르면 '재장전(reset)' 전략
```

- **실측 (10만 시드)**: 무작위 뽑기 무연결 67% → **드래프트 choose-3 무연결 29.9%**. 게다가 그 29.9%조차 "내가 일부러 끊었다"는 선택이지 무력한 사고가 아니다.
- 이 한 줄 변경이 동시에 3가지를 해결한다: (a) 죽은 탭 제거, (b) 발라트로식 "핸드에서 고르는" 주체성 회복, (c) 통함을 운이 아니라 **실력**에 종속.

### 1.2 마이크로 루프 — 카드 한 장 (0.4초 단위 손맛)

```
[행동]   3장 중 1장을 탭
   ↓ (0.15s)
[발사]   카드가 줄 끝 슬롯으로 휙 날아간다 (트윈: ease-out, 회전 살짝)
   ↓
[착지]   '척!' + 착지 먼지 파티클 + 슬롯 0.05s 눌렸다 복원
   ↓
[판정]   이웃과 색 or 숫자가 같은가? (33% — 통함은 특별 사건)
   ├─ 통함 → 번개 선 '지지직' + 칩 '+8' 팝업 + 이웃이 같은 색으로 물드는 'splash'
   └─ 끊음 → '톡' (단, 끊김도 '재장전' 부적 빌드의 트리거 — §1.5)
```

> **통함률을 33%로 낮춘 이유 (실측 기반)**: 원안 43%는 "거의 절반이 통함"이라 통함이 특별하지 않았고, 색·숫자·아이콘 3축 OR이라 "뭐가 맞았는지" 즉시 파악도 어려웠다. 2축·33%로 낮추면 **통함 = 드물고 귀한 사건** → 대비 쾌감↑, 인지도 쉬워짐. 드래프트가 그 33%를 플레이어가 끌어올린다.

### 1.3 미들 루프 — 전염 캐스케이드 (escalation 1단: 진짜 NEW 이벤트)

> **원안 최대 결함 수정**: 검증 렌즈가 "도미노는 이미 보이던 줄을 다시 세는 부기(bookkeeping)일 뿐, 베주얼드식 NEW 매치 생성이 아니다"라고 정확히 지적했다. **체인이 판을 실제로 바꾸도록 재설계했다.**

```
통함 발생 → 통한 카드가 왼쪽 이웃을 자기 색으로 '물들인다(splash)'
         → 물든 이웃이 이제 그 왼쪽과 새로 통하는가? → 새 통함 발생
         → 도미노가 아니라 '판이 변하며 새 연쇄가 태어남'
```

이건 "이미 통한 걸 거슬러 세기"가 아니라 **각 단계가 판을 바꿔 다음 통함을 *생성*하는** 진짜 캐스케이드다. 같은 7장이라도 "어디서 물들이기 시작했나"에 따라 결과가 달라진다 → **"내가 이 연쇄를 만들었다"**는 인과가 명확.

```
연쇄2 → 번개 1줄 + 색 splash 1회  | 2px shake   | 음정 기준음
연쇄3 → 번개 2줄 + splash 전파     | 4px shake   | +2반음
연쇄4 → 'tatata' 좌측 질주 + 색파동| 6px + 베이스 | +4반음
연쇄5+→ 줌펀치 + 화이트 플래시      | 8px         | +5~7반음 (귀로 상승 체감)
```

**피치 상승 = escalation 청각 신호** (한 음원 반음 시프트). 화면 상단 배율 게이지가 `x1 → x3 → x7`로 붉게 적열.

### 1.4 매크로 루프 — 라운드 정산 + **중간 폭발 분산**

> **원안 결함 수정**: "큰 폭발이 7번째 카드(라운드 끝)에 게이트돼 6탭이 무보상 도화선"이라는 지적 반영. **중간 폭발을 분산 배치.**

```
연쇄 3 도달   → 미니 폭발 #1 (화면 1/4 플래시 + 코인 짤랑)
연쇄 5 도달   → 미니 폭발 #2 (셰이크 + 슬로우모션 0.1s)
라운드 종료    → 누적 칩 × 누적 배율 대폭발 (화면 절반 'CLEAR!' 크랙 + 코인 분수)
```

라운드당 "만족스러운 폭발" 최소 2~3회를 **검증 게이트로 강제**(§9.2).

### 1.5 끊김도 보상으로 — '재장전' 빌드

드래프트로 일부러 끊는 선택이 손해만은 아니게: 일부 부적(`reload`)은 "연쇄를 끊을 때 다음 연쇄 시작 배율 +N"을 준다. → **끊기조차 전략적 도파민**. 죽은 탭이 완전히 사라진다.

### 1.6 3단 Escalation 요약

| 단계 | 무엇이 터지나 | 도파민 신호 |
|---|---|---|
| **(1) 즉발** | 고른 카드의 칩 + 색 splash | 번개 + 팝업 + '척' |
| **(2) 전염 연쇄** | splash가 만든 NEW 통함 사슬 | 피치 상승 + 게이지 적열 + 중간 폭발 |
| **(3) 메타** | 부적이 그 배율에 곱하는 폭발 | 자릿수 폭발 + 슬로우모션 + 코인 분수 |

## 2. 핵심 루프 (1판 흐름)

```
┌─────────────────────────────────────────────────────────────┐
│ [라운드 (블라인드)]                                          │
│  1. 덱 상단 3장 공개 → 1장 골라 줄 끝에 이어 붙인다           │
│  2. 통하면 즉시 칩+전염 연쇄 (splash가 NEW 통함 생성)        │
│  3. 연쇄 3·5 도달 시 중간 폭발 / 끊으면 재장전 빌드 발동      │
│  4. 9칸 다 채우면 누적 칩×배율 대폭발 → 목표선 넘으면 통과    │
└──────────────────────────┬──────────────────────────────────┘
                           ↓ 통과 시 코인 보상
┌─────────────────────────────────────────────────────────────┐
│ [상점] ← 전략 결정의 무게중심 (발라트로처럼)                  │
│  - 부적(Charm) 구매/장착 (슬롯 5칸)                          │
│  - 새 카드·각인(Brand) 추가 / 약카드 제거 / 리롤             │
│  - ★시작 끝단·덱 정렬 선택(라운드-사이 결정)                 │
└──────────────────────────┬──────────────────────────────────┘
                           ↓ 다음 라운드 (목표 급상승)
┌─────────────────────────────────────────────────────────────┐
│ [시드런 / 안테 구조]                                         │
│  안테 = 소형 블라인드 ×2 + 보스 블라인드 ×1                  │
│  보스 블라인드: 룰 변종(색맹/역순/봉인)으로 빌드 시험        │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 인-라운드 주체성 — **1개로 축소** (모순 해소)

> **원안 최대 모순 수정**: 렌즈 2·4가 공통 지적 — "탭만 하면 됨(10초 학습)" ↔ "매 카드 3결정(보류/좌우/방향)"은 양립 불가, 켜면 차가운 최적화 퍼즐(STRANDLOOM 재발), 끄면 주체성 0. 실측: 좌우 배치 전수 최적화 시 점수 +214% → 캐주얼-진지 격차 너무 큼.

**해결: 인-라운드 결정을 정확히 1개 — "3장 중 고르기" — 로 통일.**

- **드래프트(choose-3)가 곧 주체성이다**: 매 턴 "어느 카드가 연쇄를 잇나/끊나"를 고르는 단일·고임팩트 결정. 이것만으로 "내가 폭발시켰다"가 성립.
- **제거**: 좌/우 양끝 배치(×), 전파 방향 탭(×), 보류 슬롯(드래프트가 흡수). → "촤르륵 탭" 리듬 보존.
- **무게중심 이동**: 10배 격차는 인-라운드가 아니라 **상점(부적 조합)**에서 나온다. 인-라운드 결정 기여를 **±30~40%로 압축**(214% → 35%), 나머지 폭발력은 빌드로. 발라트로의 진짜 결정이 핸드 중이 아니라 샵에서 일어나는 원리.

> 결과: 캐주얼하게 골라도 70점, 잘 골라도 100점. 10배 차이는 빌드에서 → 10초 학습성과 도박감을 모두 보존, STRANDLOOM식 "매 카드 조합 숙제" 회피.

## 3. 친숙한 포맷 — 왜 10초 안에 이해되는가 (정직하게 좁힘)

> **원안 수정**: 렌즈 2가 "클론다이크(세로 더미)·화투(같은 짝) 비유는 견강부회"라 지적. **레퍼런스를 정직하게 단일화.**

| 친숙 1동작 | 누구나 아는 출처 | CHAIN DEAL에서의 모습 |
|---|---|---|
| **옆에 이어 붙인다** | **도미노 · 우노** | 줄 끝에 카드를 잇는다 |
| 같으면 통한다 | 도미노 짝, 우노 색/숫자 | 색 같음 **또는** 숫자 같음 |

**포지셔닝: "친숙한 포맷"이 아니라 "친숙한 1동작(이어 붙이기) + 새 보상(전염 연쇄 폭발)".** 우노를 안다면 통함 규칙을 0초에 안다 — 우노도 "색 또는 숫자가 같으면 낼 수 있다"이기 때문.

**진짜 한 문장 룰 (이제 거짓이 아님)**:
> 카드를 이어 붙여 옆과 **색이나 숫자가 같으면** 통한다.

- 색·숫자 **둘 다 "같으면"** — 단일 매칭 논리, 우노와 동일 직관.
- **±1 사다리 제거**: 원안의 "3은 2·4와 통하지만 3-3은 안 통함" 반직관 함정을 코어에서 삭제. (사다리의 전략 깊이는 `climber` 부적을 가진 플레이어에게만 옵트인 노출.)
- **아이콘 매칭 제거**: 통함 3축 → 2축. 아이콘은 카드의 '슈트(suit)'로 남아 부적/각인 레이어에서만 의미 (초기 인지 부하 0).

**시각적 즉시성**: 카드 = 테두리 색 + 중앙 큰 숫자. 통함 조건이 한눈에. 첫 라운드는 **색 통함만**으로 가르치고, 숫자 통함은 2라운드에 노출(점진 학습).

## 4. 새 훅 (시너지 시스템) — 클론 위험 정면 해소

> **원안 최대 클론 리스크 수정**: 렌즈 3이 "부적 = 발라트로 조커 1:1, 차별점이 한 곳에 몰림, 체인은 새 훅이 아니라 보편 콤보"라고 정확히 지적. **차별 축을 부적 밖 — '줄의 공간성' — 으로 옮긴다.**

### 4.1 세 겹 시스템 (차별 축 = 공간성)

1. **전염 체인(Contagion Chain) — 진짜 새 동사**: 통한 카드가 이웃을 **변이(색 splash/승급/와일드화)**시켜 *새* 통함을 만든다. 단순 "길수록 배율 큼"이 아니라 **판을 바꾸는 공간 연쇄**. 부적 없이 코어 룰만으로도 "이렇게 깔면 다르게 터지네"가 나온다.
2. **줄-위상 시너지(Line-Shape) — 클론과 겹치지 않는 차별 축**: 조커처럼 손패-독립이 아니라 **줄 위 위치·인접·끝단·대칭에 결부**된 부품. 예: "왼쪽 3칸 안에서만 발동", "같은 색이 연속 N칸일 때 ×", "줄이 좌우 대칭이면 폭발". → 표절 표면이 'Balatro 트리거×효과 매트릭스'에서 '줄 위상'으로 이동(겹치지 않음).
3. **부적(Charm) — 빌드 정의**: 위상 시너지를 증폭. (Balatro 조커와 구조적 유사성은 인정하되, 게임 정체성은 #1·#2에 베팅.)

### 4.2 통함(Connect) 판정 규칙 (결정론, 2축)

```typescript
connect(a, b):
  return a.color === b.color   // 색 같음
      || a.number === b.number  // 숫자 같음  (둘 다 "같으면" — 우노 직관)
  // ±1 사다리, 아이콘 매칭은 코어에서 제거 → 부적/각인 옵트인
```

### 4.3 전염 캐스케이드 (재귀 소급 → 전염 전파)

```typescript
onConnect(placed, leftNeighbor):
  // 원안 "재귀적으로 거슬러" 삭제 — 내가 안 건드린 줄 전체 재폭발은 반직관
  leftNeighbor.color = placed.color   // splash: 이웃을 물들임
  chainValue = leftNeighbor.chainValue + 1   // O(1) 이어받기
  if connect(leftNeighbor, leftNeighbor.left):  // 물듦으로 NEW 통함 생겼나?
     propagate()   // 판이 변해 새 연쇄 발생 (NEW 이벤트, 부기 아님)
```

핵심: "내가 방금 깐 카드가 이웃을 물들여 → 새 통함이 *생겼다*"는 명료한 인과. 재귀 소급의 "안 건드린 곳이 터짐" 반직관 제거.

## 5. 원자 부품 카탈로그 (data-as-code)

> **정수 고정소수점 강제**: 모든 배율은 `int×1000`. `×1.5`는 `mult += 1500` 후 정산 1회 `/1000`. 부동소수 누적 절대 금지(결정론 보장). 실측: float `1.5^7=17.0859375` ↔ fixed `17.09`, 골든 테스트·리더보드 크로스머신 일치.

## 8. 비주얼 · 연출 (정적 픽셀 + 트윈/파티클)

> 원칙: **프레임 애니 0장.** 정적 스프라이트만 GPT 생성, 움직임은 전부 코드.

### 8.1 정적 에셋 — **아트 일관성을 코드로 강제** (AI 제작가능성 수정)

> 렌즈 4 지적: "130~150개 GPT 스프라이트는 스타일 일관성이 안 나온다." **절차적 합성으로 GPT 생성량 축소.**

| 에셋 | 방식 | 비고 |
|---|---|---|
| 카드 (4색 × 9숫자) | **절차적 합성**(코드: 색+숫자 렌더) | GPT 생성 0 |
| 부적 아이콘 60~80 | **절차적**(도형+색+심볼 조합) | GPT 생성 0~소량 |
| 각인 뱃지 16~20 | 절차적 베벨 + 마스터 16색 팔레트 | 일관성 코드 강제 |
| 배경·UI·코인 | GPT 20~30개만 | 양자화 후처리 |

→ **GPT 생성 130개 → 20~30개**. 나머지는 절차적 합성 = 100% 일관 + AI 단독 부합.

### 8.2 "터지는" 연출 (코드)

| 연출 | 구현 |
|---|---|
| 카드 날기·착지 | PixiJS 트윈 (ease-out + 회전 + 먼지) |
| 번개 선 (연쇄) | 절차적 라인 + glow, 단계마다 좌측 연쇄 |
| **색 splash** | 이웃 카드 색 보간 트윈 + 파동 파티클 |
| 칩 카운트업 | 숫자 팝업 트윈 (scale up + fade) |
| 화면 흔들림 | 카메라 오프셋 트윈 (연쇄 길이 비례) |
| 중간/대폭발 | 플래시 + 슬로우모션 + 셰이크 + 코인 분수 |

### 8.3 game-feel = data-as-code (AI 튜닝 가능하게)

> 렌즈 4 핵심 지적: "game-feel 수치(셰이크 감쇠·트윈 ease·슬로우모션 길이)는 AI가 못 느끼는 암묵지." **외부화 + 인간 튜닝 마일스톤 명시.**

- 모든 game-feel 상수를 `juice.config.ts` 테이블로 외부화 (셰이크 진폭/감쇠, ease, 슬로우모션 길이, 피치 step).
- 초기값은 **레퍼런스 영상(발라트로 정산 0.X초)을 프레임 단위로 추출**해 근거 있는 시작점 + 주석.
- **M1에 "인간 1인 슬라이더 튜닝 30분 세션"을 명시적 마일스톤으로 배선** (§10). game-feel은 "AI 구현 + 인간 튜닝"임을 정직하게 인정.

### 8.4 사운드

한 음원 반음 피치 시프트로 escalation. 기본 SFX 4종: 착지('척'), 통함('지지직'), splash('촤'), 정산('쾅').

- 헤드리스 1만 시드: 죽은 탭<15%, 연쇄≥3 ≥2회/라운드, connect 30~40%, 격차<50%
- 부적 40 / 각인 14 / 블라인드 6
- 검증: 발산 0% (구조적), 난이도 곡선 분포

### M4 — 1.0 (3~4주)
- 부적 60~80 / 각인 16~20 / 블라인드 6~12
- 해금 트리 + 도전과제 + 엔드리스 + Stake 티어
- 일일 챌린지/리더보드 (정수 결정론) + 모바일 터치
- 검증: 신규 10초 룰 이해 + 60초 쾌감 + 리텐션 플레이테스트

### 11.1 정직한 미해결 가설 (플레이테스트 전엔 확정 불가)

1. **드래프트 1결정이 "충분한 주체성"인가** — 메커닉상 데드탭은 제거했으나 "재미있는 주체성"인지는 실플레이 검증 필요 (M2 게이트).
2. **전염 캐스케이드가 "놀라움"을 주는가** — NEW 이벤트 생성으로 설계했으나, 수 분 후 신선도 유지 여부는 리텐션 테스트 영역 (M4).
3. **Balatro 구조 차용의 표절 임계선** — 코어 동사(전염+위상)는 차별화했으나, 안테/블라인드/Stake 구조 유사성의 법적·체감적 임계는 출시 전 별도 판단 필요.

> 이 셋은 "수치로 못 박은 fix"가 아니라 **검증 게이트로 관리하는 잔존 리스크**다. 미달 시 밸런스 튜닝이 아니라 메커닉 재설계로 대응함을 명시한다.
