## 1. 코어 루프 & 점수 (LOCKED — 검증된 코어)

**루프**: 손패에서 카드를 골라 줄에 깐다 → 8칸 채우면 **정산(settle)**. 정산 결과가 목표 이상이면 통과(상점), 미만이면 패배(재도전권 시 재시작). 8안테 × 3블라인드(작은/큰/보스) = 24게이트.

**연결(connect)**: 인접 두 카드가 **같은 무늬 OR 같은 숫자 OR ±1 런 OR 와일드** 중 하나면 연결. (보스 룰이 일부 봉인/변형.)

**점수**:
- 카드를 깔 때 왼쪽과 연결되면 체인 길이(`runLen`)만큼 배수: `체인 점수 += sum(체인 rank) × mult`.
- `mult = runLen-1 + 부적 보정`(pyro/noir +2, suited/runner/highmult/echo +1, mult-enh 장당 +1 …), **25 캡**(발산 방지).
- 정산 시 **족보 보너스 가산**: `evalHand`의 최고 족보 1개 × `HAND_BONUS[hk] × blindBase(ante)` (가산).
  - `HAND_BONUS`: 빈도보정 — 흔한 족보 소액(투페어 .02/트리플 .05/스트레이트 .03), 희소 족보 큰 가산(플러시 .30/포카드 .50/스플 .75/파이브 .95). *8장 강제 구조라 포커 빈도 역전(플하>플러시).*
- **목표**: `blindTarget(a,b) = round( blindBase(a) × sparkComp(a) × stakeMult(a) × dmult × blindMul )`
  - `blindBase(a) = 150 × 1.5^(a-1)` · `blindMul` = 작은1 / 큰1.4 / 보스1.6 · `dmult` = 시작덱 스칼라(표준1) · `stakeMult` = 스테이크 사다리 · `sparkComp(a) = 1.0+0.34×(8-a)/7`(v3.29 불씨덱 front-loaded 보정, 초반 강·후반 무).

**★불변식(치명적)**: 모든 카드는 4더미(deck/discard/hand/row)에서 **항상 정확히 1번** 존재. 라운드전환·리롤·정산·재도전·소비아이템에서 회수 누락 금지. (리팩터서 `assertPileInvariant`로 런타임 강제.)

**LOCKED vs 튜닝**: 코어 룰·불변식 = LOCKED(재미QA로 검증). 계수(blindBase 상수·sparkComp·HAND_BONUS·mult캡)= 튜닝 포인트.

## 2. 콘텐츠 / 빌드 레이어 (24부적 → 80, 2덱 → 8)

빌드 다양성 = 리플레이 엔진. **3 연결 축**(무늬/숫자/런)이 모든 빌드의 뼈대.

- **카드 & 강화**: 시작 덱 A~8(32장) 압축 구조 유지(비대화가 연결률↓). 강화 3→**5-6종**(wild=무조건연결·gold=+base·mult=+mult + 신규 2-3종, 예: seal/edition류 — 가산·바운드).
- **시작덱 2→8**: 각 덱이 **다른 체인 전략을 가르침**(표준=균형·불씨=연결밀도 온보딩·고랭크=고점수·+플러시/짝수/소형/런지향 등 — ★표준/불씨는 현재 하나로 통합(v3.29)이라 **분리가 명시 태스크**). 인프라 = `DECKS` 레지스트리(`build`함수 + `dmult` 목표 스칼라). 파워크리프 차단 = dmult(플레이어 비가시). 상세 `2026-06-30-start-deck-variants-design.md`.
- **부적 24→80 (7-8 클러스터)**: 전부 **가산·바운드**. 클러스터 태그 + **가중 오퍼**(`CLUSTER_W`≈0.15, 미투자 클러스터 down-weight)로 희석 방지 + **리롤**(에스컬레이팅) 주체성. 80까지 확장 시 **희석 감시**(content-ledger)가 핵심(과거 13→24서 balance 9.6→3.5% 희석 선례 — 완화 레버 동반 필수). ★**희석·인플레 하드 게이트(2026-07-02)**: St0 balance 클리어율 3층 밴드 — **하드 floor ≥7.5%**(funQA P3 목표 ~9%·6.8%="부족" 판정·v3.29 달성 8.5% 정합) + **마일스톤당 하락 ≤1.0pp 래칫**(계단 침식 차단, 회귀 목표 8.5%) + **인플레 상한 기준선+2pp**(진짜 위험은 인플레 — v3.24 scout 41% 선례). 위반 = 콘텐츠 추가 보류. CLUSTER_W 재캘리브 = 마일스톤별 스윕 절차(로드맵 §3 정의, 제약 2중: G2 스프레드 + 오퍼-도달률) + **C2(24→50)가 레버 유효성 사전검증 마일스톤**(약빌드 3종 v3.29 재측정 → 버프-또는-컷 선행). 3축과 1:1 시너지(무늬≈플러시/숫자≈카르텔/런≈스트레이트).

## 3. 챌린지 레이어 (보스 12→20-22 + 소비아이템)

### 3.1 보스 (룰로 어렵게, 숫자 벽 아님)
- **액트 티어 풀**: A1(안테1-3) 순함 → A3(7-8) 가혹. 액트-final 안테(3·6·8)=고정 **액트 보스**(쇼다운, "운 좋은 순한 최종보스" 제거). tmult로 티어 보정.
- 12→**20-22**(정규 16 + 쇼다운 4-6). 각 보스 = *규칙*(무늬봉인·오름봉인·손패축소·연결세·부식…) → 플레이어가 덱 조정(적응). 신규 보스도 **룰 기반**, tmult는 run-sim 풀런 캘리브(단일라운드 balance-check 아님).
- ★**축 커버리지 요구(2026-07-02, C7)**: 현 12종은 rank축=mono·run축=mono+seal_climb 커버, **suit축 실질 카운터 0**(seal_suit=♠ 1/4만) + 안테8 anchor는 가산 레이어(족보+Current) 무접촉. → C7 확장 시 ①3축 각각 최소 1개 봉인/과세 보스, ②쇼다운 풀에 **가산 감쇠 룰 1종**(정산 가산 ×0.5 — 곱 신설 아닌 가산의 바운드). 매트릭스·후보 룰 상세 = 소비아이템 스펙 §4.4.
- ★보스 룰이 **소비아이템 Currents의 자연 카운터**(축봉인 보스 = 한 축 몰빵 천적). 상세 `2026-06-29-position-boss-rule-design.md`.

### 3.2 소비아이템 (신규 3티어 — 요약, 상세 스펙 별도)
제3 레이어(charm=패시브·enh=카드부착과 비중복). **슬롯 3 · 하이브리드 타이밍.**
- **Splices**(구제/유틸, 라운드중, ~16): 카드/줄 잔손질로 체인 구제(수비적 주체성 → 즉사 완화).
- **Currents ★시그니처**(축 성장, 상점, ~12): 발라트로 행성의 CHAIN판 — 무늬/숫자/런 **연결 축을 영구 레벨업**(가산·이중캡, 정산 시 그 축 연결마다 보너스↑). "내 빌드축에 투자".
- **Surges**(증폭/도박, 라운드중, ~10-12): 극적 스윙(공격적 클라이맥스 → 도파민).
- ★상세·수식·카운터플레이 = **`docs/superpowers/specs/2026-07-01-consumable-system-design.md`** (본 문서는 요약·중복 금지).

## 4. 진행 & 경제 (스테이크 6→8 · 언락/도전과제/챌린지 신설)

- **마스터리 격자 = 덱(8) × 스테이크(8) = 64셀** → 100h+ 꼬리(콘텐츠 저비용, 곱연산 깊이).
- **스테이크 0-5 → 0-7**: `stakeMult`(STAKE_T/STAKE_AC 가산 델타) + baseHand/보스 델타. St0 기준선(현 ~8.5%) 불변 가드 → 최상위 단조 하락 >0. ★`handBonus`/broker/twins는 `blindBase`(스테이크 무관) 사용 = 목표만 오르고 보너스 불변(상쇄버그 차단). 상세 `2026-06-25-difficulty-ladder-design.md`.
- **경제**: `goldEarned = floor(GOLD_BASE(1) + (점수/목표−1)×GOLD_K(4))`. 상점 티어가격(8/5/3) + 에스컬레이팅 리롤 + 가중 오퍼. 런 종료 `spillover = floor(남은골드×0.1)` → `cd_meta.coins`. 메타 상점(`cd_meta`: 재도전권/시작골드Lv/리롤Lv). 불변식 = `economy-check.cjs`.
- **신규 언락 트리(~65-75)**: 콘텐츠를 게이팅해 100h 아크 페이싱. 언락 트리거 = **가르치는 플레이 목표**("무늬-런너로 승리", "25 mult 체인 달성"). 미플레이 아키타입 미해금 = 상점 풀서 제외(희석 완화 부수효과).
- **도전과제 ~45**(Steam 업적/Google PGS 필수 자산 겸용) · **챌린지 모드 12-15**(고정 시드/변형 룰).

## 5. 프레젠테이션 (★픽셀 아트 · 절차적 합성 — 초기 결정 LOCKED)

- **스타일 = 정적 픽셀 아트**, 프레임 애니 0장(움직임은 전부 코드). GDD §8/§9.4 결정.
- **인게임 = 절차적 합성(코드, GPT 0)**: 카드/부적 80/enh/보스 20 = 도형+심볼+클러스터 팔레트 조합 렌더 → **100% 일관**(마스터 16색). ★핵심 아트-엔지니어링 태스크 = **절차적 아이콘 생성기** + 변별력 검증(80종 컨택시트 구분 패스).
- **GPT = 배경/UI ~20-30개만** + 16색 양자화(일관성). AI 도구 = GPT(초기 결정), AI 노출 최소.
- **주스/오디오(기존 강점 — 보존·확장)**: sparkBurst 파티클·shake/flash·revealTally 순차 카운트업(목표돌파 클라이맥스)·WebAudio 절차 사운드(beep/boom). ★트레일러 소재로 이미 우수.
- **UI/UX**: `render()`가 DOM 재구성(상태변화 시만·호버 재렌더 금지). 모바일 반응형(v3.19-20, 8칸 scale-to-fit). 스토어 히어로(캡슐/트레일러) = 픽셀 스타일, 외주/집중제작.
- **접근성(신규 — 가독성=흐름 축)**: 색이 게임플레이 정보를 나른다(빨강♥♦/검정♠♣ 부적 pyro/noir/투톤·red_curse 보스, 클러스터 액센트=가독성). ①**마스터-16 팔레트의 색맹-안전 검증**(deuter/protan/tritan 시뮬 + 대비)을 **A0 팔레트 태스크에 포함**(그때 하면 비용 ~0), ②무늬는 글리프(♠♥♦♣)로 이미 형태 구분 — 색 의존 정보(클러스터 액센트·빨강/검정)에 **모양/패턴 이중화 옵션**, ③텍스트 스케일 옵션. 스토어 접근성 태그 대응.
- **온보딩(신규 — fun QA P1 직결)**: **pre-v3.29 진단**(judgment-2026-07-01, 당시 캐주얼 안테1 93% 즉사)이 "온보딩 = 대중재미를 죽이는 1차 원인"으로 지목 → v3.29 spark가 **구조**(연결밀도↑, 80% PASS)는 완화했으나, 진단이 노출한 **인간 이해 격차는 봇이 측정하지 못하는 잔존 영역** — 튜토리얼 = 이해 완화, 상보적이며 spark가 대체하지 않음. **인터랙티브 첫 런**: ①connect 3룰(놓을 곳 하이라이트 유도) → ②정산/족보 → ③상점/부적 → ④소비아이템(C4 이후) → ⑤보스 적응, 규모 = 안테1 1회분 가이드 + 신규 요소 첫 조우 컨텍스트 팁 1회. ★**반증 레버 준수: 난이도를 깎는 온보딩(순수 초반완화) 금지 — 가르치는 온보딩만**(측정요약: 완화는 캐주얼 fun을 오히려 낮춤). 검증 = sim 불가 영역 → 인간 플레이테스트 지표(첫 세션 완주율/이탈 시점 — **기준선은 v3.29 빌드로 신규 측정**해 설정). 마일스톤 = **C9**(~2주, 로드맵 §3).

## 6. 플랫폼 & 배포 (백엔드리스 — 총 현금 ~$125)

- **Steam(PC) = Electron + `steamworks.js`**(Tauri 아님 — Node 컨텍스트 필요). 도전과제·리더보드·클라우드세이브 = **Valve 호스팅(자체 서버 0)**. $100 Direct.
- **Google Play(Android) = Capacitor**(TWA 아님, 단일 로컬 HTML 래핑), **target API = 제출 시점 Play 플로어**(연 +1 상향 — 2027 출시면 ≥API 36 예상, .aab 빌드 달에 재확인), Play Games Services(백엔드리스). $25.
- **세이브**: `platform/storage` 추상화(localStorage → Steam Cloud / Android) + **스키마 버저닝(신규)**: `cd_meta`/`cd_stats`/`cd_unlocked`에 `schemaVer` 필드 + 전방 마이그레이션 사다리(v→v+1) — C2~C8에서 스키마가 계속 변형(소비템 슬롯·스테이크 8·언락 노드)되므로 Phase 0 storage 설계에 포함(라이브 세이브 생긴 뒤 retrofit은 고통). 클라우드 충돌 룰 = **higher-progress-wins**(로그라이크 메타에 newest-wins보다 안전). **리더보드**: 플랫폼 호스팅(위조 가능 — 캐주얼 보드는 수용). **데일리 경쟁 보드**: 결정론적 시드+리플레이 재검증($0 Cloudflare Worker, 기존 `LOG_URL` 재활용) = 옛 드리프트 고통을 안티치트 오라클로 전환.
- **컴플라이언스(신규 — 하드 제출 블로커)**: ①Play **개인정보처리방침 URL + Data-Safety 폼 필수** — 게임이 이미 `LOG_URL`로 pid+**유저 닉네임**+점수 전송(코드의 "개인정보 X" 주석은 부정확: 닉네임=유저 데이터) → 전송 항목 신고 또는 모바일서 LOG_URL 게이트오프 결정. ②Steam **AI-disclosure 설문** = GPT 에셋 1개라도 필수(노출 "최소"와 무관). ③IARC(무료·즉시)는 기존 §6 계획 유지.
- **수익**: Steam 유료 선불 / Play 유료언락 IAP(또는 보상형 광고 — 공정성 게이팅 금지). ★**마케팅 = Coming-Soon 페이지 Day 1**(위시리스트 = 성공 결정자). ★Play 하드월: 신규 개인계정 12테스터×14일. 상세 `production-roadmap.md §5`.

## 7. 아키텍처 & 밸런스 툴링 (Phase 0 — 모든 콘텐츠의 선행조건)

- **모듈/데이터주도 리팩터**: 단일 `index.html` → `src/`(`rules`/`content`/`engine`/`ui`/`platform`) + **esbuild → 단일 배포 HTML**(무런타임뎁 유지) + 바닐라 JS. Strangler-fig(매 스텝 배포가능).
- **★i18n 문자열 외부화 = Phase 0 스텝(신규)**: 현재 `index.html`에 한국어 `name:`/`desc:` 리터럴 ~46개 — Phase 0가 바로 그 데이터 객체를 전부 옮기므로 **그때 key-out**(content 데이터에 키, 텍스트는 로케일 JSON — KR+EN 최소). 지금 안 하면 80부적+40소비템에 인라인 한국어 실린 뒤 **2번 지불**(전 플랜 최대 비용의 rework). 도전과제 45·언락 트리거 문구 = 스토어 노출 텍스트라 i18n 필수 자산.
- ★**미러 드리프트 영구 제거**: 부적/보스/덱 = **효과 레지스트리 훅 데이터** 1객체. 게임과 sim이 **동일 rules/content 모듈** 호출(`run-sim`은 얇은 어댑터) → "콘텐츠 추가 = 4파일 편집" → "**1객체**".
- **밸런스 툴 스케일 = `gate.cjs`**: 한 커맨드 회귀 게이트(카드 불변식·빌드 다양성 스프레드·조건부클리어 flatness·경제 불변식·fun-축 비회귀·3뱅크 강건성) + **content-ledger**(희석/파워크리프 트렌드 감시) + 소비아이템 사용 페르소나. 콘텐츠 추가마다 자동 검증. 상세 `production-roadmap.md §2-3`.

- **소비아이템**: Current 변형 목록·계수, Surge 다운사이드 강도, Splice 카드추가 허용 범위(불변식). (설계 스펙 튜닝 포인트. 슬롯 수=3·범람 룰·축 카운터 요구는 **2026-07-02 확정** — 스펙 §3/§4.4/§6.)

## 9. 빌드 시퀀스 (로드맵 연결)

**Phase 0**(리팩터 ~2주, i18n key-out 포함) → **C1**(gate.cjs 툴링) → **C2-C9 콘텐츠**(부적→덱→소비아이템→보스→스테이크/언락→**온보딩**, 매 단계 gate + 플레이테스트 통과) ∥ **아트 A0-A4**(절차적 생성기→80아이콘→배경→히어로/트레일러) → **Phase B**(Steam+Google 래핑·서비스·스토어·컴플라이언스). 마케팅 Coming-Soon = Day 1. 총 **~9-14개월(솔로)** — 단 아트∥콘텐츠는 솔로에겐 인터리빙(직렬 합산 풀타임 ~8-10개월, 로드맵 §1 정직 노트). 상세 `production-roadmap.md`.

## 10. 검증

전 게임 밸런스/재미/경제는 sim 툴체인으로 게이트: `gate.cjs`(회귀 종합) · `run-sim.cjs`(조건부 클리어 flatness·스테이크 사다리·덱 스윕·경제) · `funqa`(5페르소나 대중재미 ≥70% + 3뱅크 강건) · `economy-check.cjs`(경제 불변식) · `balance-check.cjs`(문법+단일라운드) · 소비아이템 페르소나/지표. **콘텐츠는 gate green + 인간 플레이테스트(최종 권위) 통과 후에만 병합.**

**★재미 게이트 상세 (SSoT = `tools/funqa/measurement-summary-2026-07-01.md`, 2026-07-02 명문화)**:
- **판정**: 5페르소나(마스터리·안전제일·콤보러·스릴러·캐주얼) fun ≥6.0 · 대중재미 ≥70% · **3 시드뱅크(50000/70000/90000, N=2000) 전부 — 최저 뱅크 기준**(평균 아님; 캐주얼 칼날-위 재발 방지). 회귀 가드 = 페르소나 fun 하락 **≤0.2**(G6, SSoT 재QA 게이트 값) — 초과 하락은 v3.29 safe −0.35처럼 **명명·기록된 수용 결정**으로만 출시 가능.
- **수용된 예외(봇 아티팩트 — 게이트 무관)**: 스릴러 ~5.98 영구 임계미달, 콤보러 정량/정성 괴리(픽 아티팩트). 단 **소비아이템(C4/C5)이 이 둘의 지정 개선 레버**(Splices→near-miss 완충, Surges→tension/도파민 스파이크) — 재QA 시 개선 방향 확인, 미달해도 블록 아님.
- **반증 레버 재도입 금지**: 순수 초반완화(relief) · 긴체인 점수보너스(opt1) — 측정요약 §6서 기각 · 균일 blindBase 보정 — HANDOVER §6 v3.29 캘리브서 기각(flatness 붕괴, 안테8 63%).
- **정량 지표 사용 원칙**: 하한(FAIL) 탐지에 강, 상한 해석에 약 — 상위 페르소나 점수 무비판 신뢰 금지. **인간 = 상한(진짜 손맛) 판정자** = "최종 권위"의 운영적 정의. 공수 = 마일스톤당 ~15% + Phase B 전 밸런스-QA 1-2주(로드맵 §3).
