## 1. 목표 & 배경

**무엇**: 발라트로 Stakes / 슬더슬 Ascension 식 **누적 난이도 등급제**. 이전 스테이크를 **클리어로 해금**, 각 스테이크가 게임에 **작은 규칙 하나를 누적**으로 더한다. 같은 게임을 점점 빡세게 다시 도는 구조.

**왜 (최고 ROI 장수 레버)**: 새 콘텐츠(카드·보스·아트) 0개로 **있는 콘텐츠 전부(보스12·부적10·안테8)를 곱한다.** 티어 하나 = 한 줄 규칙 델타(저작비용 ≈ 0)인데 24블라인드 전체에 작용 → 리플레이가 통째로 늘어남. 단일 HTML·솔로 개발에 가장 가성비 좋은 깊이 추가. 리더보드에 **난이도 축**도 생김.

**측정된 출발점 (★캘리브의 핵심 근거)**: 2026-06-25 헬스체크 — 조건부 클리어율(도달자 중 통과%)이 **전 블라인드 평탄·고율 85~97%**. 전체 클리어 ~9%는 **24블라인드 게이트 곱연산**(0.9²⁴)에서 나옴. **유일한 실제 벽 = 안테8 닻(anchor, mult cap 3) ~68%.** → 사다리는 이 *이미 존재하는* 곡선을 **균일하게 아래로 밀어** 난이도를 올려야 하며, 새 분산을 주입하면 안 됨.

## 2. 핵심 설계 결정 (WHY)

| 결정 | 이유 |
|---|---|
| **점수-스칼라(ST)를 척추로** | 측정상 점수가 목표 바로 위에 몰림 → 작은 목표 상승이 조건부 클리어를 크게 깎음(분산 0 추가). `blindTarget`에 `ST` 한 변수 = 가장 예측가능·캘리브 쉬움, 미러 동기화도 순수함수라 최易. |
| **경제·규칙 텍스처 혼합** | 7티어가 전부 "목표 +X%"면 단조로움. 경제(통과골드 압박)·규칙(보스전 손패 2장, 보스룰 확장)으로 축을 섞어 매 티어가 *다르게 느껴지게*. |
| **가산>곱셈 100% 보존** | 모든 델타는 목표·캡·보너스·경제·규칙 — **체인 점수식(×엔진)은 일절 불변.** raw 점수 곱셈 금지(HANDOVER §3.2 "아슬아슬" 파괴 방지). |
| **스킬 게이팅(P2W 아님)** | 티어는 *클리어로만* 해금. 메타 골드로 못 삼. 자발적 챌린지. |
| **고위험 규칙 티어 컷/보류** | 전역 ±1 봉인·이중보스는 과다하락 + connect 3파일 드리프트 + 보스쌍 호환성(1줄 초과). v1에서 제외, "극한 티어"로 보류. |
| **시작자원 삭감 컷** | 경제팀 T5/T6(시작 retry/gold −) = 고인물 *투자 무력화* 불만 + cashOut의 `meta.retry=S.retry` 경로 드리프트. 통과골드 압박만 채택. |
| **닻 보스는 캡 드래그서 제외** | 캡 25→20 전역 적용하되 **닻은 3 유지**(2로 안 내림) — 이미 유일한 벽인 안테8을 더 높이면 난이도가 거기 *재집중*. 난이도는 *분산*시켜야 곡선이 매끄럽게 내려감. |

## 3. 티어 래더 (권장안 — 스테이크 0~7)

> 수치는 **캘리브 목표**(전체 그리디 클리어율). 정확값은 `run-sim` 조건부-클리어로 **누적**(티어 N = 1..N) 튜닝. 각 델타는 `S.stake` 단일 전역 의존 → cumulative 자동 성립.

| St | 이름 | 누적 수정자 | 함수 | 목표 클리어 | 축 |
|---|---|---|---|---|---|
| 0 | ⚪ **평지** | (현재 그대로) | — | ~9% | 기준 |
| 1 | 🔩 **백금** | 모든 목표 `×(1+0.06·stake)` → +6% | `blindTarget` | ~6% | 점수 |
| 2 | 🪙 **인색** | 통과 골드 바닥 제거 `GOLD_BASE 1→0`(초과분만 지급) | `goldEarned` | ~4.5% | 경제 |
| 3 | ⚙ **강철** | 후반 가속: 목표에 안테 비례 항 추가(stake≥3) | `blindTarget` | ~3.3% | 점수 |
| 4 | ❄ **서리** | 보스 블라인드 손패 3→2 (모든 액트보스, 'stingy' 일반화) | `startBlind` | ~2.5% | 규칙(에이전시) |
| 5 | ⚓ **닻줄** | 체인 배율 캡 25→20 (★닻 보스는 3 유지) | `placeCard` | ~1.8% | 점수(천장) |
| 6 | 👁 **그림자** | 보스 *룰*이 큰 블라인드에도 적용(tmult는 제외 — 순수 규칙) | `startBlind` | ~1.2% | 규칙(정체성) |
| 7 | 🕳 **심연** | 목표 가속 강화: ST 계수 0.06→0.08 | `blindTarget` | <1% | 점수(정점) |

**티어별 메모 (구현·캘리브 주의):**
- **St1 백금**: `const base=150*Math.pow(1.5,ante-1)` → `* (1+0.06*S.stake)`. 0.06 계수를 조절해 조건부 중앙값 ~−5pp/블라인드.
- **St2 인색**: `goldEarned=(s,t)=>Math.floor(GOLD_BASE_eff + max(0,s/t-1)*GOLD_K)`에서 `GOLD_BASE_eff = stake>=2 ? 0 : 1`. ★`run-sim` 5빌드 비교로 **약빌드도 가끔은 클리어**(빌드 viability 사수, 봉쇄 금지) 확인.
- **St3 강철**: ST에 `+ 0.04*max(0,stake-2)*(ante-1)` 추가 → 안테1 거의 무변, 안테8 대폭. 조건부 곡선이 안테8로 *완만히* 하강(절벽 아님)하게 0.04 튜닝(안테8 조건부 ~60~65 목표).
- **St4 서리**: `baseHand = (boss&&boss.id==='stingy')?2 : (S.blind===2?2:3)`. ★sim은 *에이전시 손실*(수읽기 −33%)을 점수로만 봐 **과소평가** → sim 하락폭(~5pp)보다 체감이 큼. sim 수치만 보고 더 쌓지 말 것. 손패확장 스킬이 명시적 카운터.
- **St5 닻줄**: `Math.min(mult, (boss&&boss.id==='anchor')?3:(stake>=5?20:25))`. `hand-frequency.cjs`로 mult≥20 체인이 실제로 자주 나오는지(캡이 *물리는지*) 확인 — 드물면 18로.
- **St6 그림자**: `S.boss=(S.blind>=1&&S.ante>=2)?S.anteBoss:null`(안테1 큰블라인드는 상점 전이라 면제) **+ tmult 가드를 `if(S.boss&&S.blind===2)`로** (큰블라인드는 룰만, 목표 경감 없음). 배너도 큰블라인드서 보스 표시. connect/placeCard는 이미 S.boss를 읽으므로 *추가 변경 없음*.
- **St7 심연**: St1의 0.06을 0.08로. **이론상 클리어 가능 유지**(0% 벽이면 0.07로 후퇴) — 사다리는 *축*이지 벽이 아님.

**보류(극한 티어, v2+)**: 전역 ±1 봉인(connect 3파일 드리프트 + 과다하락), 이중보스(보스쌍 호환 블랙리스트 필요=1줄 초과). 드리프트 작업 감수 시 별도 스펙.

## 4. 아키텍처

- **런 상태**: `S.stake`(0~maxStake). `newGame(seed, stake)`에서 설정. 모든 델타가 이 전역을 읽음.
- **메타 저장**: `cd_meta.maxStake`(해금된 최고 스테이크, 기본 0). 런 종료 시 **그 스테이크로 클리어(승리)** 하면 `maxStake = max(maxStake, stake+1)`. `getMeta/saveMeta` 확장(기존 META_DEFAULT에 `maxStake:0` 추가).
- **해금 흐름**: `victory()`에서 `if(S.stake>=meta.maxStake) meta.maxStake=S.stake+1; saveMeta()`. (스테이크 N 클리어 → N+1 해금.)
- **시뮬 파라미터**: `run-sim.cjs runFull(strat, acc, stake)` — stake 인자 추가, 델타 미러에 동일 적용. 캘리브 = 각 스테이크를 누적으로 돌려 목표 클리어율 맞춤(조건부 곡선 도구 활용).

## 5. UI / UX

- **스테이크 선택**: 새 게임 영역에 칩 `스테이크 N ◀▶`(0~maxStake). `newGame(null, stake)` / 데일리는 `newGame(dailySeed(), stake)`. 잠긴 스테이크는 회색+자물쇠.
- **현재 런 표시**: HUD에 `St N` 배지(안테 칩 옆). 정산·배너에 스테이크 명칭.
- **해금 알림**: victory 시 "🔓 스테이크 N+1 해금!" (정산 unlock 알림 패턴 재사용).
- **각 스테이크 설명**: 메타/도움말 드로어에 티어별 한 줄 규칙 목록(무엇이 추가되는지 투명 공개).

## 6. 리더보드 통합

- **점수에 스테이크 동봉**: `submitScore(runBest, S.stake)` → 시트 레코드에 stake 컬럼. ★규칙이 바뀌므로 **스테이크 다른 점수는 비교 불가** — 반드시 같이 저장·표시.
- **표시**: 리더보드 각 행에 `[St N]` 태그. (v1: 태그만. v2: 스테이크별 탭/필터, 또는 "데일리 @ 고정 스테이크" 경쟁.)
- **데일리**: 데일리를 *고정 스테이크*(예: 그 주의 권장 스테이크)로 돌리는 모드 고려 — 공정 비교 + 난이도 경쟁. (선택, v2.)

## 7. 메타-파워 vs 스킬 (정책)

- 사다리는 **스킬 게이팅**(클리어로 해금). 메타 골드로 스테이크 못 삼.
- **재도전권 가드**: 상위 스테이크(예: ≥5)에서 `useRetry` 1줄 가드로 **재도전 비활성** → 메타 retry로 고스테이크 brute-force 방지(스킬 순수성). (선택 — 1줄, `if(S.stake>=5) return;`.)
- 시작 골드/리롤 메타 성장(goldLv/rerollLv)은 **삭감하지 않음**(투자 무력화 불만 회피) — 대신 통과골드 압박(St2)으로 간접 조임.

## 8. 캘리브레이션 계획

1. `run-sim.cjs`에 `stake` 인자 배선(델타 미러 포함).
2. **누적** 측정: 각 스테이크 0~7을 `runFull(...,stake)`로 5빌드 + 조건부-클리어 곡선 출력.
3. 각 티어 계수를 **목표 클리어율(§3 표)** 에 맞춤. 특히:
   - 조건부 곡선이 *절벽 아닌 완만한 하강* 유지(특정 블라인드 0% 금지).
   - **≥2 빌드 viability** 사수(St2 경제 봉쇄 주의).
   - 최고 스테이크도 **이론상 클리어 가능**(>0%).
4. `balance-check.cjs`는 단일 라운드라 **경제 티어(St2) 측정 불가** → run-sim이 권위. balance-check 미러는 문법·기준선 가드.

## 9. 드리프트 동기화 지점 (★CRITICAL)

각 티어 델타를 **`index.html` ↔ `run-sim.cjs` ↔ `balance-check.cjs` 3곳** 미러:

| 티어 | index.html | run-sim.cjs | balance-check.cjs |
|---|---|---|---|
| St1/3/7 | `blindTarget` | `blindTarget` | `blindTarget` |
| St2 | `goldEarned` | `goldEarned` | (경제=무관, 미러 불필요) |
| St4 | `startBlind` baseHand | `runFull` handN | (단일라운드—선택) |
| St5 | `placeCard` mult cap | `gain` mult cap | `gain` mult cap |
| St6 | `startBlind` boss/tmult | `runFull` boss/target | (단일라운드—선택) |

→ 구현 시 **티어 하나씩** 추가+3파일 동기화+조건부 곡선 재측정. 드리프트 가드: 스테이크 0이면 모든 델타가 no-op(기준선 9% 불변) 확인.

## 13. 구현 결과 (2026-06-25 캘리브 — 설계 대비 변경)

구현 = v3.22. **계획 7스테이크 → 실측 후 6스테이크(St0~5)로 축소** + 레버 재배치. 이유는 캘리브로 드러난 사실들:

- ★**게이트 곱연산의 좁은 범위**: 전체 클리어 = per-blind 통과율^24. 기준선 per-blind ~0.91(→9%). 사용 가능 범위는 per-blind **0.91~0.80(약 11pp)** 뿐 — 0.80이면 이미 ~0.4%. 즉 7티어로 <1%까지 매끄럽게 내리려면 티어당 per-blind ~1.5pp의 *극히 미세한* 레버가 필요. 디스크릿 레버(골드 −2.2%p, 손패2 −1.4%p, 보스룰큰 −0.7%p)는 **너무 거칢** → 후반 2티어가 0%로 붕괴. → **6스테이크가 정직**(각 티어 뚜렷·전부 클리어 가능).
- **배율 캡 레버(25→20) 제외**: 밸런스 빌드에서 mult≥20 체인이 드물어 **사실상 무효**(측정: −0%p). 코드 캡은 25 유지.
- **보스룰-큰블라인드 레버 보류**: 바닥(~1%)에서 −0.7%p가 0% 붕괴를 만듦. 코드는 유지하되 `STK>=6`으로 **6스테이크 래더선 비활성**(향후 base 상승/7+ 확장 시 활성).
- **시작자원 삭감(계획 St5/6) 컷**: 미구현(고인물 투자 무력화 불만 + cashOut 드리프트 회피).
- ★**보너스 기준 분리 버그 차단**: `handBonus`/broker/twins가 `blindTarget(ante,0)`을 쓰면 목표 인상이 족보 보너스 쿠션도 키워 난이도가 상쇄됨 → **`blindBase(ante)`(스테이크 무관)로 교체**.

**최종 6스테이크(run-sim 밸런스 클리어율)**: St0 8.9% / St1 백금 6.6%(목표+3%) / St2 인색 3.9%(통과골드 바닥0) / St3 강철 2.4%(후반 가속) / St4 서리 1.0%(보스전 손패2) / St5 심연 0.6%(목표 추가가속). 델타·UI·드리프트·검증은 HANDOVER §6 참조.

**남은 것**: 리더보드 *표시* stake 태그(백엔드 Apps Script stake 컬럼 필요, 클라는 전송 중). 보류 극한 티어(전역 ±1 봉인·이중보스)는 별도 스펙 시 v2.
