# 소비아이템 시스템 설계 (Consumables) — CHAIN DEAL

> 작성: 2026-07-01 · 상태: **설계 확정(구현 전)** · 스코프: 콘텐츠 마일스톤 C4/C5 (production-roadmap §3)
> 선행 컨텍스트: `docs/production-roadmap.md`(§3 콘텐츠·§8 미결정) · `tools/funqa/measurement-summary-2026-07-01.md`(재미 진단)
> ★수치(계수·가격·레벨캡)는 전부 **sim 캘리브 대상** — 본 스펙은 *설계 의도 + 튜닝 포인트*만 고정한다.

---

## 1. 동기 (왜)

재미QA 종합 진단(측정요약 §2 실측 블로커: 캐주얼 도파민 밀도·near-miss)에서 도출한 설계 해석으로, CHAIN DEAL의 구조적 공백은 **"moment-to-moment 의사결정 레이어 부재"**다(★이 문구는 본 스펙의 종합이지 QA 문서의 직접 인용이 아님). 현재 두 콘텐츠 레이어 —
- **charm** = 영구 패시브(런 내내 자동 발동),
- **enh** = 카드 부착 수식어(wild/gold/mult),
— 는 둘 다 *수동적*이다. 플레이어가 *지금 이 순간* 능동적으로 내리는 전술 결정이 없다.

**소비아이템 = 제3 레이어**: 능동 사용 + (시그니처) 연결 축 성장. 발라트로의 타로/행성/스펙트럴이 채우는 자리를, CHAIN 코어(연결)에 맞게 재해석한다.

## 2. 설계 목표

1. **순간 주체성**: 라운드 중 능동 사용으로 즉석 전술 결정을 만든다(측정요약 직접 대응).
2. **CHAIN-네이티브**: 범용이 아니라 코어 행동(connect/체인/줄)을 레버로 쓴다.
3. **가산·바운드**: 프로젝트 밸런스 법칙(가산 > 곱셈, mult-25 캡) 준수 — 소비아이템도 발산 금지.
4. **레이어 비중복**: charm(패시브)·enh(카드부착)와 기능이 겹치지 않는다.
5. **기존 시스템 재활용**: 신규 보스/코어룰 추가 없이 기존 축봉인 보스·charm과 시너지/카운터.

## 3. 시스템 스켈레톤

**3티어 · 슬롯 3개 · 하이브리드 타이밍.**

| 티어 | 성격 | 타이밍 | 위험 | 희소도 | 개수(출시 목표) |
|---|---|---|---|---|---|
| **Splices** | 구제/유틸 (카드·줄 잔손질) | 라운드 중 + 상점 | 저 | 흔함 | ~16 |
| **Currents** ★시그니처 | 연결 축 영구 성장 | **상점 전용**(투자) | — | 중 | ~12 |
| **Surges** | 증폭/도박 (극적 스윙) | 라운드 중 + 상점 | 고 | 희소 | ~10-12 |

- **슬롯 3개**: 초과 획득 시 취사선택(사용 또는 판매) 압박. 소지 = 슬롯에 홀드. ★**3 확정 근거(fun, 2026-07-02 재검토)**: 3티어가 각기 다른 fun축 담당(Splice=캐주얼 생존/near-miss 완화 · Current=마스터리 투자 · Surge=도파민 클라이맥스) → **티어당 1홀드**가 가능해야 티어-역할 분화가 성립. 2슬롯이면 구제 vs 클라이맥스 택일 강제 → **캐주얼 블로커**(측정요약 §2-B: dopamine 0.52·0-spike 25%·death 14%) 대응 설계 약화. 스릴러는 v3.29서 봇 아티팩트로 수용된 상태 — Surge의 tension 기여는 '가능성'으로만 취급(성립 근거 아님). 튜닝 사다리 = 아래로만 — 단 하향 조건은 2중: ①C4 sim 비축/데드웨이트 잉여 + ②**G6(페르소나 fun 하락 ≤0.2)·G7(3뱅크 최저뱅크 대중재미 ≥70%) 통과 시에만**(슬롯 수 = 캐주얼 구제 용량에 직결이라 하향 자체가 fun 회귀 리스크).
- **획득**: 상점 오퍼(골드, 티어별 가격 `Splice < Current < Surge`) + 가끔 블라인드/보스 보상 드롭.
- **타이밍 규칙**: Currents = **상점에서만**(성장 투자는 라운드 사이 결정). Splices·Surges = **라운드 중(카드 배치 도중)** + 상점.

## 4. ★Currents (시그니처) — 연결 축 성장

### 4.1 코어 메커닉
발라트로 **행성 카드**(= 족보 하나를 영구 레벨업)의 CHAIN판. CHAIN의 코어는 *연결*이므로 **연결 축(axis)을 레벨업**한다.

- 연결 축 = **무늬(suit) / 숫자(rank) / 런(±1)** 3개. 각각 플러시·카르텔·스트레이트 빌드와 1:1.
- 상점에서 Current 사용 → 해당 축 **영구 +1레벨**(그 런 내내). 스택 가능, **최대 레벨 캡**(≈8, sim 튜닝).
- "내 빌드가 쓰는 축에 투자해 키우는" 성장 엔진 → 후반 빌드 스케일.

### 4.2 정산 수식 (가산·바운드)
```
정산(settle) 시, 축 X가 레벨 L(L≥1)이면:
  axisConn = 최종 줄에서 X로 연결된 인접쌍 수
             ★단일 정의(2026-07-02 확정): 연결 성립 경로(와일드/범람 오버라이드 포함)와
               무관하게 **인접쌍의 원값(실제 무늬/숫자/±1) 일치로만** 판정. 한 쌍이 복수 축
               충족 시 각 축에 카운트. "와일드 연결 미포함" = 와일드로만 성립한 연결(원값
               무일치 쌍) 미포함이라는 뜻 — 와일드 **카드**도 원값을 보유하므로 우연 일치 시
               카운트됨(무마킹 카드와 동일 기여 = 인플레 0). ★카드-레벨 제외(와일드 낀 쌍
               무조건 미포함)는 기각: 불씨덱(=캐주얼 덱) 와일드 4장이 Currents와 안티시너지
               가 되는 자기모순. connect가 wild를 단락 처리하므로 axisConn은 settle서 원값
               독립 계산 = 구현 자연스러움(evalHand의 wild 제외 카운팅도 무훼손).
  보너스 += round( blindBase(ante) × K × L × min(axisConn, CONN_CAP) )
```
- **K ≈ 0.02, CONN_CAP ≈ 6** (튜닝 포인트).
- **가산**(체인 ×엔진 미경유 → 분산 폭발 없음), `blindBase(ante)` 비례(후반 relevance, 스테이크 무관 — handBonus/broker 선례), **L·conn 이중 캡**으로 바운드.
- handBonus와 동일하게 정산 단계 가산(placeCard 체인 점수와 별도 레이어).
- 예시(감): 무늬 L4·안테3·무늬연결 5 → `round(337.5 × 0.02 × 4 × 5) ≈ 135` 가산 — 유의미하나 비폭발.

### 4.3 Current 풀 구성 (~12, 3축 × 변형)
3축뿐이므로 "12 distinct"는 **base + 변형**으로 채운다(발라트로 행성이 반복 등장하며 스택하듯):
- **코어 3종**: 무늬/숫자/런 축 +1 (반복 획득으로 스택).
- **변형 ~9종**(수집·전략 다양성): 예 — *대류*(한 축 +2, 고가) / *분류*(두 축 +1씩, 약하게) / *프리즘류*(3축 소량) / *적응류*(이번 판 가장 많이 쓴 축 +1) / *즉발류*(+1 레벨 + 그 자리 즉시 소량 페이아웃).
- 변형 목록·계수는 C5 구현 시 sim으로 확정.

### 4.4 기존 시스템 시너지·카운터플레이 (★2026-07-02 재검토 — 코드 대조 정정)

**현 12보스 축-커버리지 매트릭스 (코드 사실)**:

| 축 | 실질 카운터 보스 | 커버 |
|---|---|---|
| 숫자(rank) | `mono`(같은 숫자 연결 무효) | ✅ 완전 |
| 런(±1) | `mono`(연속 무효 = 완전) + `seal_climb`(오름만 무효 = 부분·그리디 자동 우회, v3.26 표적검증 델타 +5pp) | ✅ 1.5개 |
| **무늬(suit)** | `seal_suit`는 **♠만** 봉인(1/4) — ♥/♦/♣ 무늬 빌드 무접촉. `red_curse`는 기본점수 과세라 **연결(axisConn)은 유지** = Currents 카운터 아님. `mono`는 오히려 무늬 빌드를 **도움** | ❌ **실질 0** |

- **정정**: 종전 "seal_suit·seal_climb = 한 축 몰빵의 천적" 서술은 suit축에 대해 부정확. 현 풀에서 **suit축 Current 몰빵의 구조적 카운터가 없다**. 추가로 안테8 고정 쇼다운 `anchor`는 체인 mult만 캡(3) — **정산 가산 레이어(족보+Current)에 무력**.
- **→ C7 보스 확장 요구사항 (구조 보장)**: ①**축 커버리지 매트릭스 충족** — 3축 각각 최소 1개 정규 보스가 해당 축을 봉인/과세, **우선순위 = 비♠ suit축**(유일 공백). 후보 룰: 적응형 봉인(★**봉인 축 = 라운드 시작 시 고정** — 직전 라운드 최다 사용 축 기준, 보스 배너로 사전 공지. 라이브 재판정 금지: 라운드 중 판정이면 기배치 체인이 소급 단절·⚡예고 어긋남(v3.26 예고 회귀와 동형). 구현 = seal_suit/mono처럼 **connect 내 대칭 룰**(placeCard 훅 불필요). 동률 타이브레이크 = suit>rank>run 고정(데일리 결정론 보존)), 무늬세(suit 연결마다 소액 감점). ②쇼다운 풀(4-6종)에 **가산 감쇠 룰 1종**(정산 가산 보너스 ×0.5) — 클라이맥스에서 가산 스택 빌드도 체인 본체로 승부하게. (곱 신설 아님 — 가산의 감쇠 = 바운드, 가산>곱셈 법칙 유지. toll bonus×0.5·dull −1·anchor 캡3 선례와 동형, 스테이크 비결합 = 상쇄버그 없음.)
- 이중캡(L·conn)이 규모를 바운드하고, 축 카운터 보스는 **적응 압력**을 담당(둘은 보완 관계 — 캡만으로는 "몰빵=무전략 최적"을 못 막음).
- **시너지**: `mono` 보스(무늬 빌드 유리), `suited`/`runner`/`pyro`/`noir` charm이 축과 결합.
- **와일드 분리**: spark 덱/enh 와일드 **카드**는 축-중립(§4.2) → Currents를 인플레하지 않음(깨끗한 경계). (범람의 판정 오버라이드와 구분 — §6.)
- **검증 게이트(§10 연동)**: 3축 몰빵 봇 각각 클리어율 스프레드 연속 + **표적 조건부 통과율**(★측정 주체 = 해당 축 몰빵 봇 vs 그 카운터 보스 — run-sim `seal_climb` 표적검증 블록 패턴) 55~85% 밴드. balance 봇 기준 전 보스는 **별도 유지** — 관측 범위 78~98%(측정요약), 게이트 밴드 72~98%(로드맵 G3 baseline-anchored)·벽≤1·G4 flatness. 두 프레임 혼동 금지. 신규 카운터 보스는 기존 유일 벽(안테8 닻 ~69%) 이상으로 캘리브. ★선행 작업(C1): run-sim STRATS에 **순수 런축 몰빵 봇 부재**(현 11종 중 flush=suit·cartel=rank만 존재) — 런축 strat 1종 추가 필수. 신규 보스는 tension 축 직결이므로 **gate.cjs G6(페르소나 fun 하락 ≤0.2)·G7(3뱅크 최저뱅크 대중재미 ≥70%)** 비회귀도 게이트에 포함.

## 5. Splices (Tier1 — 구제/유틸, 라운드 중, ~16)

끊긴·나쁜 상황을 **잔손질로 구제**하는 저위험 능동 도구. **수비적 순간 주체성** → 캐주얼·스릴러 즉사/near-miss 완화(측정요약 두 블로커 중 하나에 기여).

★**카드 불변식(4더미 합 32) 보존**: **기존 카드 수정 위주**(무늬/랭크/enh/위치). 카드 추가/삭제는 최소화하고, 허용 시 4더미 총량을 정확히 추적(invariant 사수).

| 예시 | 효과 | 불변식 |
|---|---|---|
| 재배선 | 손패/줄 카드 1장 **무늬 변경** | 수정(안전) |
| 눈금이동 | 카드 1장 **랭크 ±1** | 수정(안전) |
| 각인 | 카드 1장 **와일드화**(enh) | 수정(안전) |
| 자리바꿈 | 줄의 두 카드 **위치 스왑** | 수정(안전) |
| 땜질 | 인접 두 카드 **강제 연결 1회**(브릿지) | 판정만(안전) |
| 재드로우 | 손패 버리고 새로 | 더미이동(회수 사수) |

## 6. Surges (Tier3 — 증폭/도박, 라운드 중, ~10-12)

**판을 뒤집는 큰 스윙**, 일부는 다운사이드 동반(고위험 도파민). **공격적 클라이맥스** → 도파민 스파이크(측정요약 캐주얼 도파민 축과 스릴러 tension에 기여). ★전부 가산/바운드.

| 예시 | 효과 (가산/바운드) |
|---|---|
| 과부하 | 이번 정산, 최고 레벨 축 Current 보너스를 **한 번 더 가산**(실질 2배 — 단 이미 L·conn 캡으로 바운드된 값의 1회 배수라 발산 없음, 체인 ×엔진 미경유) |
| 역류 | 줄 순서 **반전**(재배열로 대형 체인 성립) |
| 범람 | **사용 시점 이후의 배치**에서 연결 판정·runLen 재평가에 전부-성공 오버라이드 적용(판정 오버라이드 — 카드가 와일드가 되는 게 아님). ★**기적립 체인 점수 소급 재계산 없음** — 체인 점수는 placeCard 배치 시점마다 증분 적립되므로(발라트로와 다름, HANDOVER §4) "정산 시점 일괄 적용"은 no-op이거나 의도 초과: **사용 타이밍 = 스킬 결정**이 되어 Surge 취지(순간 주체성)와 정합. ★**axisConn은 카드 원값 기준 유지**(§4.2 단일 정의 — 원값 일치 쌍만, 와일드-단독 성립 쌍 미포함) → Currents 투자를 0화하지 않음(2026-07-02 확정: 와일드 축-중립 룰을 범람에 적용하면 시그니처 투자 배반 = 콤보러 "재밌게 지기" 함정과 동형 → 데드웨이트·타이밍분산 악화). 과부하와 조합 = **의도된 하이롤**. 실 바운드 = **SLOTS 8**(runLen≤8 → 기본 mult≤7, 부적 풀스택 ≈+13로 25캡 내); mult-25 캡은 백스톱(종전 "mult-25 캡이 자연 바운드"는 오식별 정정). ★잭팟 부적(runLen≥4 시 체인 보너스 ×2, mult 캡 외부 경로)은 범람 시 상시 발동 — 실효 상한 = 런 rank합(≤64) × mult(≤25) × 잭팟2 (× 보스 감쇠), C5 캘리브에 포함 |
| 도박 | 목표 **≥1.3배** 통과 시 대박(골드/보너스), 미달 시 **벌칙**(카드/골드 손실) |
| 용융 | 카드 1장 **파괴 → 대형 1회 보너스**(덱 압축 트레이드) |

## 7. 데이터 스키마 (Phase 0 리팩터 정렬)

각 소비아이템 = 데이터 1객체 + 효과 훅(효과 레지스트리 패턴):
```
{ id, name, tier: 'splice'|'current'|'surge',
  cost, timing: 'shop'|'mid'|'any',
  target: 'card'|'row'|'axis'|'self'|'none',
  axis?: 'suit'|'rank'|'run',        // Currents 전용
  apply(state, ctx) { ... }          // 순수 상태 변환 (engine/consume 슬라이스)
}
```
- 게임·sim이 **동일 모듈**(engine/consume + rules) 공유 → 미러 드리프트 0(리팩터 성과 상속).
- Currents의 정산 훅은 `rules/scoring`의 settle 파이프라인에 등록.

## 8. 밸런스 가드 + sim 필요 (신규)

- **가산-only 법칙**: Surges 포함 전 티어 바운드. 범람 = 판정 오버라이드(§6) — 실 바운드는 **SLOTS 8**(runLen≤8 → 기본 mult≤7)이고 mult-25 캡은 부적 스택 백스톱(+잭팟 ×2는 캡 외부 기존 경로, §6 상한식 참조).
- **Currents 이중 캡**(L·conn) + **기존 축봉인 보스 카운터**로 몰빵 억제 → sim 튜닝(K·L캡·CONN_CAP·가격).
- **신규 sim 페르소나**(로드맵 §3/C4 반영): 소비아이템 사용 성향 봇 — 비축가/낭비가/최적화/패닉.
- **신규 지표**: 클러치 가치(구제로 살린 판), 타이밍-분산(*항상 같은 타이밍에 쓰이면 오토파일럿 → 컷*), 데드웨이트(안 쓰이는 소비아이템).
- **회귀 게이트**: `gate.cjs`(C1 툴링 — Phase 0=C0 리팩터 후 별도 마일스톤) — 카드 불변식·빌드 다양성 스프레드·조건부클리어 flatness·경제 불변식·fun-축 비회귀·3뱅크 강건성.

## 9. 스코프 경계 (YAGNI)

- **포함**: 3티어, 3축 Currents(코어 3 + 변형), 슬롯 3, 하이브리드 타이밍, 상점/드롭 획득, 위 예시 규모.
- **제외(후속/보류)**: 소비아이템 자체의 해금 게이팅(콘텐츠 페이싱 단계서), 소비아이템 전용 보스, 4번째 축/메타축, 소비아이템 조합(craft) 시스템.

## 10. 검증 (구현 후)

- `gate.cjs` 전 항목 green (신규 소비아이템 페르소나·지표 포함). ★전 판정 = **3 시드뱅크(50000/70000/90000) 최저 뱅크 기준**(측정요약 재QA 게이트 — 평균 아님).
- Currents: 단일 축 몰빵이 지배 전략이 되지 않음 — **3축 몰빵 봇 각각** 클리어율 스프레드 연속 + 축별 카운터 보스(§4.4 C7 매트릭스) 앞 조건부 통과율 55~85%.
- Splices/Surges: 데드웨이트 0(전부 유의미), 타이밍-분산 > 임계(오토파일럿 아님).
- 재미 회귀: 캐주얼 near-miss/도파민 개선 방향 + **스릴러 tension 개선 방향**(Surges가 지정 레버 — 수용된 봇 아티팩트라 미달해도 블록 아님, 방향만 확인), 마스터리/콤보 fun 하락 ≤0.2.
