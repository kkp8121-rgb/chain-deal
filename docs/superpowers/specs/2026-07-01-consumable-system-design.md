# 소비아이템 시스템 설계 (Consumables) — CHAIN DEAL

> 작성: 2026-07-01 · 상태: **설계 확정(구현 전)** · 스코프: 콘텐츠 마일스톤 C4/C5 (production-roadmap §3)
> 선행 컨텍스트: `docs/production-roadmap.md`(§3 콘텐츠·§8 미결정) · `tools/funqa/measurement-summary-2026-07-01.md`(재미 진단)
> ★수치(계수·가격·레벨캡)는 전부 **sim 캘리브 대상** — 본 스펙은 *설계 의도 + 튜닝 포인트*만 고정한다.

---

## 1. 동기 (왜)

재미QA 종합 진단이 CHAIN DEAL의 구조적 공백으로 **"moment-to-moment 의사결정 레이어 부재"**를 짚었다. 현재 두 콘텐츠 레이어 —
- **charm** = 영구 패시브(런 내내 자동 발동),
- **enh** = 카드 부착 수식어(wild/gold/mult),
— 는 둘 다 *수동적*이다. 플레이어가 *지금 이 순간* 능동적으로 내리는 전술 결정이 없다.

**소비아이템 = 제3 레이어**: 능동 사용 + (시그니처) 연결 축 성장. 발라트로의 타로/행성/스펙트럴이 채우는 자리를, CHAIN 코어(연결)에 맞게 재해석한다.

## 2. 설계 목표

1. **순간 주체성**: 라운드 중 능동 사용으로 즉석 전술 결정을 만든다(측정요약 직접 대응).
2. **CHAIN-네이티브**: 범용이 아니라 코어 행동(connect/체인/줄)을 레버로 쓴다.
3. **가산·바운드**: 프로젝트 밸런스 법칙(가산 > 곱셈, mult-25 캡) 준수 — 소비아이템도 발산 금지.
4. **레이어 비중복**: charm(패시브)·enh(카드부착)와 기능이 겹치지 않는다.
5. **기존 시스템 재활용**: 신규 보스/코어룰 추가 없이 기존 축봉인 보스·charm과 시너지/카운터.

## 3. 시스템 스켈레톤

**3티어 · 슬롯 3개 · 하이브리드 타이밍.**

| 티어 | 성격 | 타이밍 | 위험 | 희소도 | 개수(출시 목표) |
|---|---|---|---|---|---|
| **Splices** | 구제/유틸 (카드·줄 잔손질) | 라운드 중 + 상점 | 저 | 흔함 | ~16 |
| **Currents** ★시그니처 | 연결 축 영구 성장 | **상점 전용**(투자) | — | 중 | ~12 |
| **Surges** | 증폭/도박 (극적 스윙) | 라운드 중 + 상점 | 고 | 희소 | ~10-12 |

- **슬롯 3개**: 초과 획득 시 취사선택(사용 또는 판매) 압박. 소지 = 슬롯에 홀드.
- **획득**: 상점 오퍼(골드, 티어별 가격 `Splice < Current < Surge`) + 가끔 블라인드/보스 보상 드롭.
- **타이밍 규칙**: Currents = **상점에서만**(성장 투자는 라운드 사이 결정). Splices·Surges = **라운드 중(카드 배치 도중)** + 상점.

## 4. ★Currents (시그니처) — 연결 축 성장

### 4.1 코어 메커닉
발라트로 **행성 카드**(= 족보 하나를 영구 레벨업)의 CHAIN판. CHAIN의 코어는 *연결*이므로 **연결 축(axis)을 레벨업**한다.

- 연결 축 = **무늬(suit) / 숫자(rank) / 런(±1)** 3개. 각각 플러시·카르텔·스트레이트 빌드와 1:1.
- 상점에서 Current 사용 → 해당 축 **영구 +1레벨**(그 런 내내). 스택 가능, **최대 레벨 캡**(≈8, sim 튜닝).
- "내 빌드가 쓰는 축에 투자해 키우는" 성장 엔진 → 후반 빌드 스케일.

### 4.2 정산 수식 (가산·바운드)
```
정산(settle) 시, 축 X가 레벨 L(L≥1)이면:
  axisConn = 최종 줄에서 X로 연결된 인접쌍 수
             (와일드 연결은 축-중립 → 미포함; 한 쌍이 복수 축 충족 시 각 축에 카운트)
  보너스 += round( blindBase(ante) × K × L × min(axisConn, CONN_CAP) )
```
- **K ≈ 0.02, CONN_CAP ≈ 6** (튜닝 포인트).
- **가산**(체인 ×엔진 미경유 → 분산 폭발 없음), `blindBase(ante)` 비례(후반 relevance, 스테이크 무관 — handBonus/broker 선례), **L·conn 이중 캡**으로 바운드.
- handBonus와 동일하게 정산 단계 가산(placeCard 체인 점수와 별도 레이어).
- 예시(감): 무늬 L4·안테3·무늬연결 5 → `round(337.5 × 0.02 × 4 × 5) ≈ 135` 가산 — 유의미하나 비폭발.

### 4.3 Current 풀 구성 (~12, 3축 × 변형)
3축뿐이므로 "12 distinct"는 **base + 변형**으로 채운다(발라트로 행성이 반복 등장하며 스택하듯):
- **코어 3종**: 무늬/숫자/런 축 +1 (반복 획득으로 스택).
- **변형 ~9종**(수집·전략 다양성): 예 — *대류*(한 축 +2, 고가) / *분류*(두 축 +1씩, 약하게) / *프리즘류*(3축 소량) / *적응류*(이번 판 가장 많이 쓴 축 +1) / *즉발류*(+1 레벨 + 그 자리 즉시 소량 페이아웃).
- 변형 목록·계수는 C5 구현 시 sim으로 확정.

### 4.4 기존 시스템 시너지·카운터플레이 (★신규 보스/룰 0)
- **카운터**: `seal_suit`(무늬봉인)·`seal_climb`(오름 ±1 봉인) 보스 = 한 축 몰빵의 천적 → 축 분산/우회 적응 강제. 한 축 올인이 자동으로 최적이 되지 않게 하는 **내장 안전판**.
- **시너지**: `mono`(단색) 보스, `suited`/`runner`/`pyro`/`noir` charm이 축과 결합.
- **와일드 분리**: spark 덱/enh 와일드는 축-중립(§4.2) → Currents를 인플레하지 않음(깨끗한 경계).

## 5. Splices (Tier1 — 구제/유틸, 라운드 중, ~16)

끊긴·나쁜 상황을 **잔손질로 구제**하는 저위험 능동 도구. **수비적 순간 주체성** → 캐주얼·스릴러 즉사/near-miss 완화(측정요약 두 블로커 중 하나에 기여).

★**카드 불변식(4더미 합 32) 보존**: **기존 카드 수정 위주**(무늬/랭크/enh/위치). 카드 추가/삭제는 최소화하고, 허용 시 4더미 총량을 정확히 추적(invariant 사수).

| 예시 | 효과 | 불변식 |
|---|---|---|
| 재배선 | 손패/줄 카드 1장 **무늬 변경** | 수정(안전) |
| 눈금이동 | 카드 1장 **랭크 ±1** | 수정(안전) |
| 각인 | 카드 1장 **와일드화**(enh) | 수정(안전) |
| 자리바꿈 | 줄의 두 카드 **위치 스왑** | 수정(안전) |
| 땜질 | 인접 두 카드 **강제 연결 1회**(브릿지) | 판정만(안전) |
| 재드로우 | 손패 버리고 새로 | 더미이동(회수 사수) |

## 6. Surges (Tier3 — 증폭/도박, 라운드 중, ~10-12)

**판을 뒤집는 큰 스윙**, 일부는 다운사이드 동반(고위험 도파민). **공격적 클라이맥스** → 도파민 스파이크(측정요약 캐주얼 도파민 축과 스릴러 tension에 기여). ★전부 가산/바운드.

| 예시 | 효과 (가산/바운드) |
|---|---|
| 과부하 | 이번 정산, 최고 레벨 축 Current 보너스를 **한 번 더 가산**(실질 2배 — 단 이미 L·conn 캡으로 바운드된 값의 1회 배수라 발산 없음, 체인 ×엔진 미경유) |
| 역류 | 줄 순서 **반전**(재배열로 대형 체인 성립) |
| 범람 | 이번 정산 줄 전체 **와일드 취급**(전부 연결 → mult-25 캡이 자연 바운드) |
| 도박 | 목표 **≥1.3배** 통과 시 대박(골드/보너스), 미달 시 **벌칙**(카드/골드 손실) |
| 용융 | 카드 1장 **파괴 → 대형 1회 보너스**(덱 압축 트레이드) |

## 7. 데이터 스키마 (Phase 0 리팩터 정렬)

각 소비아이템 = 데이터 1객체 + 효과 훅(효과 레지스트리 패턴):
```
{ id, name, tier: 'splice'|'current'|'surge',
  cost, timing: 'shop'|'mid'|'any',
  target: 'card'|'row'|'axis'|'self'|'none',
  axis?: 'suit'|'rank'|'run',        // Currents 전용
  apply(state, ctx) { ... }          // 순수 상태 변환 (engine/consume 슬라이스)
}
```
- 게임·sim이 **동일 모듈**(engine/consume + rules) 공유 → 미러 드리프트 0(리팩터 성과 상속).
- Currents의 정산 훅은 `rules/scoring`의 settle 파이프라인에 등록.

## 8. 밸런스 가드 + sim 필요 (신규)

- **가산-only 법칙**: Surges 포함 전 티어 바운드. "범람=전체 와일드"는 기존 mult-25 캡이 상한을 강제.
- **Currents 이중 캡**(L·conn) + **기존 축봉인 보스 카운터**로 몰빵 억제 → sim 튜닝(K·L캡·CONN_CAP·가격).
- **신규 sim 페르소나**(로드맵 §5 반영): 소비아이템 사용 성향 봇 — 비축가/낭비가/최적화/패닉.
- **신규 지표**: 클러치 가치(구제로 살린 판), 타이밍-분산(*항상 같은 타이밍에 쓰이면 오토파일럿 → 컷*), 데드웨이트(안 쓰이는 소비아이템).
- **회귀 게이트**: `gate.cjs`(Phase 0 툴링) — 카드 불변식·빌드 다양성 스프레드·조건부클리어 flatness·경제 불변식·fun-축 비회귀·3뱅크 강건성.

## 9. 스코프 경계 (YAGNI)

- **포함**: 3티어, 3축 Currents(코어 3 + 변형), 슬롯 3, 하이브리드 타이밍, 상점/드롭 획득, 위 예시 규모.
- **제외(후속/보류)**: 소비아이템 자체의 해금 게이팅(콘텐츠 페이싱 단계서), 소비아이템 전용 보스, 4번째 축/메타축, 소비아이템 조합(craft) 시스템.

## 10. 검증 (구현 후)

- `gate.cjs` 전 항목 green (신규 소비아이템 페르소나·지표 포함).
- Currents: 단일 축 몰빵이 지배 전략이 되지 않음(축봉인 보스 앞에서 붕괴) + 3축 클리어율 스프레드 연속.
- Splices/Surges: 데드웨이트 0(전부 유의미), 타이밍-분산 > 임계(오토파일럿 아님).
- 재미 회귀: 캐주얼 near-miss/도파민 개선 방향(측정요약 축), 마스터리/콤보 fun 하락 없음.

## 튜닝 포인트 요약 (sim 캘리브 대상)
`Current K(≈0.02)` · `CONN_CAP(≈6)` · `Current 레벨캡(≈8)` · 티어별 가격·드롭률 · Surge 다운사이드 강도 · Splice 카드추가 허용 범위 · 변형 Current 계수.
