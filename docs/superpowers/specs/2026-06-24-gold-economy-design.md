# 골드 경제 + 아웃게임 메타 (Gold Economy) — 설계

> 최종 업데이트: 2026-06-24 · 대상 버전: v3.16 예정
> PLAN.md **C2(아웃게임 경제)** 의 본 구현. HANDOVER §7 🥉 "아웃게임(메타) 층 — 초과 보상 1/10 정산".
> ⚠️ 이 스펙은 앞서 논의된 "무료 상점 + 초과분→메타코인" 안을 **대체**한다 (사용자 2026-06-24 방향 전환: 인게임 골드 경제 신설).

## 0. 한 줄 요약

지금까지 **무료(3장 중 1장 공짜 픽)** 이던 인게임 상점을 **골드 유료 상점**으로 바꾼다. 블라인드를 통과하면 **초과분이 골드로 환전**되고, 그 골드로 **가격이 매겨진 스킬**(강스킬 비쌈/약스킬 쌈)을 산다. 런 종료 시 **남기고 안 쓴 골드의 1/10** 이 아웃게임 메타 화폐로 반출되어, 메타 상점에서 **재도전권·시작 골드·시작 리롤**을 산다.

## 1. 새 루프 / 핵심 텐션

```
[런 내] 블라인드 통과 → 초과분을 골드로 환전
          → 골드로 상점 스킬 구매(유료, 강=비쌈/약=쌈) → 남은 골드 다음 블라인드로 이월
[런 종료] 최종 승리 OR 패배 → 남은 골드 × 1/10 → 아웃게임 메타 화폐
[아웃게임] 메타 화폐로 재도전권 / 시작 골드+N / 시작 리롤+1 구매
```

신규 의사결정 2축:
- **소비 vs 비축**: 이번 런에 골드를 써서 강해지기 vs 남겨서 1/10 메타 반출.
- **집중 vs 분산**: 비싼 강스킬 1개 vs 싼 약스킬 여러 개.

## 2. ⚠️ 최대 리스크 — 밸런스 기준선 이동 (이 작업의 절반)

현재 무료 획득률(상점당 1품목 픽)이 **밸런스 기준선의 핵심 축**이다 (그리디 클리어 작은~91%/큰~81%/보스55~71% — `balance-check.cjs` 캘리브). 유료화하면 **구매 품목 수가 줄어 게임이 어려워진다.**

→ **설계 목표**: 환전 레이트 × 가격을 **"중앙값 플레이어가 상점당 ≈1품목 구매"** 로 맞춰 **현 획득률을 보존**. 잘하면 초과분↑→골드↑→2개(보상), 못하면 0~1개. **차등이 곧 보상.** 무료게임의 기준 클리어율을 ±몇 %p 안에서 유지하는 것이 DoD(§8).

## 3. 골드 획득 — 초과율 환전 (A안 채택)

초과분을 **절대값이 아니라 비율**로 환전 → 안테 1~8 내내 골드가 일정 밴드 → 가격 평탄 유지:

```js
// 블라인드 통과(settle pass) 시:
const over = Math.max(0, S.score / S.target - 1);     // 초과율 (0=간당간당, 0.5=50% 초과)
S.gold += Math.floor(BASE + over * K);
```

- **A안 채택 (소액 base + 초과율)**: `BASE`로 간당간당 통과도 약간 벌어 **데스나선 방지**(진행 엔진 유지 = 게임 코어 원칙). 사용자 워딩 "남은 점수를 골드로"에서 소폭 확장.
- **B안 (순수 초과, BASE=0)**: 워딩 충실·"압도해야 부유" 극대화하나, 빠듯하게 이기는 플레이어가 스노볼 탈락 → 스윙↑. **리뷰에서 B 선택 시 `BASE=0`로 한 줄 변경.**
- **초기값(sim 캘리브 대상)**: `BASE=2`, `K=10`. 예: 50% 초과 → `floor(2+5)=7`골드, 간당간당 → `2`골드.

## 4. 인게임 상점 — 유료화 + 가격 티어

### 4.1 구조 변경 (`openShop`/`chooseShop`)
- 현재: 3장 제시 → 1장 **무료 픽** or 스킵 → `advanceBlind`.
- 신규: 3장 제시 + **각 가격표** → 골드로 **살 수 있는 만큼**(0~3장) 구매, 살 때마다 골드 차감·해당 카드 비활성(품절). 다 사거나 그만두면 "상점 나가기 →" → `advanceBlind`.
- 보유 골드 상시 표시. 골드 부족 품목은 비활성(흐림).
- 상점 리롤(새로고침 비용)·이자는 **비범위**(§7).

### 4.2 가격 티어 (좋은/나쁜 차등 — 초기값, sim 대상)
| 티어 | 가격 | 품목 |
|---|---|---|
| **상** | 8 | 핵심 부적(greed·pyro·noir·jackpot·suited·runner) · **와일드 부여 ★**(무엇과도 연결=최강) · 손패확장 +1 |
| **중** | 5 | 배율석 ◆ · 황금 ● · 리롤 +1 · 카드 복제 ⧉ · 보조 부적(broker·twins·compactor·runts) |
| **하** | 3 | 덱 압축 ✂ · 카드 추가 +(덱 비대화 리스크라 저렴) |

- 가격은 `shopPool()` 각 품목에 `cost` 필드로 부여. 정확 숫자는 §3의 `BASE/K`와 함께 **sim 캘리브로 동시 확정**.

## 5. 아웃게임 메타 (런 종료 스필오버)

- **스필오버**: 런 종료(`settle`의 런-종료 분기 = 패배 또는 최종 보스 클리어) 시 `meta.coins += floor(S.gold * 0.1)`.
- **메타 화폐**: `localStorage["cd_meta"].coins`.
- **메타 상점 3종** (드로어 `openDrawer('meta')`, 기존 드로어 패턴 재사용):
  | 상품 | 타입 | 효과 | 캡 | 초기 가격(메타코인) |
  |---|---|---|---|---|
  | 🎟 재도전권 | 소모품(반복) | 블라인드 실패 시 그 라운드 재시도 | 보유 3 | 3 |
  | 💰 시작 골드 +N | 영구 Lv1~3 | 런 시작 골드 +(Lv×3) | Lv3 | 5 / 8 / 12 |
  | 🔄 시작 리롤 +1 | 영구 Lv1~2 | 런 시작 `rerollMax` +Lv | Lv2 | 6 / 10 |
- 메타코인은 천천히 쌓이고(남은 골드의 1/10) 인게임 소비와 경쟁하므로 메타 가격은 소액. 모두 sim/플레이테스트 대상.

### 5.1 재도전권 작동 (메커니즘 검증 완료)
패배 정산 시점엔 카드가 이미 discard로 회수된 상태(`settle`의 `S.discard.push(...S.row); S.discard.push(...S.hand)`). 따라서 재시도 =
```js
S.deck = shuffle(S.deck.concat(S.discard)); S.discard = [];   // 전량 deck 회수
S.over = false; S.settled = false;                            // 런 종료 취소
startBlind();                                                 // 같은 ante/blind 재시작 (S.target·boss 동일 재산출)
```
- **★카드 불변식(치명)**: 회수 후 4더미 합이 정확히 현재 덱 장수와 일치해야 함(HANDOVER §3.2, CLAUDE.md 불변식). row/hand는 이미 비어 있으므로 deck+discard 전량 회수면 안전.
- 보스는 `S.anteBoss`라 재선택 없이 유지. 골드는 보존(재도전권이 보상이므로 골드 환전은 재시도 클리어 시 정상 발생).
- 사용 UI: 패배 정산 표(`#tally`)에 `S.retry>0`이면 "🎟 재도전권 사용 (재시도)" 버튼 추가, 클릭 시 `S.retry--` + 위 로직.

## 6. 데이터 / 영속성

- **런 상태**: `S.gold`(신규 필드, `newGame`에서 초기화), `S.retry`(보유 재도전권, `newGame`에서 `meta.retry` 로드).
- **메타**: `localStorage["cd_meta"]` = `{coins, retry, goldLv, rerollLv}`, 기본 `{coins:0,retry:0,goldLv:0,rerollLv:0}`. `getMeta()/saveMeta()` (기존 `getStats/saveStats` 패턴 복제).
- **`newGame()`**: `S.gold = meta.goldLv*3`; `S.rerollMax = meta.rerollLv`(기존 0 대체); `S.retry = meta.retry`.
- **`settle()` 훅**: (1)통과 시 골드 환전(§3), (2)런-종료 분기에서 스필오버 적립(§5) + `saveMeta`.

## 7. 비범위 (YAGNI)

- 상점 리롤(새로고침 비용) · 이자(interest) · 라이프포인트 · 랜덤 시작 부적 — 차기.
- 메타 상품은 위 3종으로 고정. 추가 상품·아웃게임 트리(스킬트리) 등은 별개.

## 8. 검증 기준 (Definition of Done)

1. **sim 갱신(가장 무거움)**: `run-sim.cjs`에 골드 경제(획득 §3·구매 의사결정·가격 §4) 모델링, `balance-check.cjs`에 골드 게이팅 반영. CLAUDE.md "규칙 중복 드리프트" 준수.
2. **기준선 보존**: 캘리브 후 그리디 클리어율이 무료게임 대비 ±몇 %p 내(작은~91%/큰~81%/보스55~71% 근방). 벗어나면 `BASE/K`·가격 재튜닝.
3. **문법/회귀**: `node tools/balance-check.cjs` 문법 OK.
4. **신규 테스트** `tools/economy-check.cjs`: 골드 환전 공식, 스필오버 1/10, 재도전권 재시도 후 **카드 불변식(4더미 합 = 덱 장수)** 검증.
5. **신규 플레이어(빈 localStorage)**: 시작 골드 0·리롤 0·재도전권 0. 상점 유료 작동, 골드 부족 시 구매 불가.
6. **메타 루프**: 런 종료 시 남은 골드 1/10 적립 → 메타 상점에서 재도전권/시작골드/시작리롤 구매 → 다음 런에 반영.
7. **재도전권**: 패배 시 버튼으로 재시도, 같은 덱·보스로 블라인드 재시작, 카드 불변식 유지.
