## 1. 게임이 무엇인가 (룰)

- 화면 중앙에 **8칸 줄**. 아래 **손패 3장** 중 하나를 클릭해 줄에 왼→오로 깐다.
- 깐 카드가 **바로 왼쪽 카드와 연결**되면 체인 발생:
  - **같은 무늬(♠♥♦♣)** OR **같은 숫자** OR **±1 연속 숫자** → 연결 (포커의 플러시·페어·스트레이트 직관)
- 연결이 **연속으로 길수록 배율 ↑** → 점수 폭발. 카드 **숫자 = 기본 점수값**(항상 획득 + 매칭에도 사용).
- 8장 다 깔면 정산 → **목표 점수(블라인드)** 넘기면 통과.
- **런 구조**: 안테 1~8, 각 안테 = `작은 → 큰 → 보스` 블라인드. 블라인드마다 **덱빌딩 상점**.
- **보스 블라인드** = 특수 규칙으로 전략 방해(아래 §5). **안테 8 보스 격파 = 런 클리어 = 승리.**
- **죽음 = 처음부터 재시작** (로그라이크. 발라트로식 목표-미달-사망 모델).

## 4. 기술 아키텍처

- **단일 파일**: `prototype/index.html` — HTML + CSS + vanilla JS 전부 인라인. 빌드/번들/프레임워크 없음. 외부 라이브러리 0.
- **상태**: 전역 `S` 객체 하나 (ante, blind, deck, discard, row, hand, score, target, boss, owned, bonusHand, rerollMax, …).
- **렌더**: `render()`가 DOM 전체 재구성(상태 변화 시에만 호출). **호버는 클래스 토글만**(과거 호버-재렌더가 클릭 씹힘 버그를 냈음 — 절대 호버에서 render() 부르지 말 것).
- **연출(juice)**: 정적 카드 + CSS 트윈/애니메이션 + 파티클(spark) + 화면 흔들림/플래시 + **WebAudio 절차 사운드**(체인마다 음정 상승). 프레임 애니 0장. ← "손맛"의 핵심. 건드릴 때 주의.
- **사운드**: `beep(freq,dur)` 오실레이터, `boom(f)` 저음 임팩트.

### 점수 공식 (정확히)
```
카드 = {suit:0~3(♠♥♦♣), rank:1~8, enh:null|'wild'|'gold'|'mult'}
connect(a,b): wild이면 true / 보스 seal_suit면 ♠ 불가 / 그 외: 같은무늬 || 같은랭크 || (|Δrank|==1 && 보스 seal_run 아님)

카드 1장 깔 때:
  base = (보스 red_curse && 빨강) ? 0 : rank
  base += 3 (탐욕 부적) ; base += 5 (gold 효과)
  gained = base
  연결되면:
    runLen = 새 카드에서 왼쪽으로 연속 연결된 길이
    mult = runLen - 1
    +2(발화:빨강) +1(플러시광:같은무늬) +1(러너:±1) +1(런 안의 mult효과 카드마다)
    보스 dull이면 mult = max(1, mult-1)
    mult = min(mult, 25)   ← 발산 방지 캡
    sum = 런에 포함된 카드들의 rank 합
    bonus = sum * mult ; 잭팟부적 && runLen>=4 면 bonus *= 2
    gained += bonus

최종 정산(settle, 8장 완성 시 — 카드별 누적 score 위에 한 번):
  족보 = evalHand(8장)                                   # 텍사스 서열 최고 1개
  족보보너스 = round(blindTarget(ante,0) * HAND_BONUS[족보])  # ★가산(곱 아님 = 아슬아슬 보존)
  score += 족보보너스  →  score >= target 이면 통과
```

### 핵심 파라미터 (밸런스 조정 지점)
- 시작 덱: A~8, 4무늬 = 32장 (`fullDeck()`)
- 손패: 기본 3 + `bonusHand` (스킬), 인색한손 보스 = 2
- 라운드 = 8칸, 안테 = 8개, 블라인드 = 3개(작은/큰/보스)
- 목표: `blindTarget(ante,blind) = round(150 * 1.5^(ante-1) * [작은1 / 큰1.4 / 보스1.6])` × 보스 `tmult`
- 보스 tmult: 코드 `BOSSES` 배열(12종, 0.4~1.0 · 룰 가혹할수록 낮음) = SSoT — §5 발췌/§6 참조
- 보스 등장: `pickBoss(ante)`가 액트 풀(A1=안테1-3 / A2=4-6 / A3=7-8)에서 선택, 액트-final 안테(3·6·8)는 고정 액트보스. **안테1 풀 = 단색저주·무딘칼날·보릿고개(act1 비-final 3종).** (구 minAnte 게이팅은 v3.22 액트 시스템으로 폐기.)
- 족보 보너스 계수 `HAND_BONUS`: 페어0 / 투페어.02 / 트리플.05 / 스트레이트.03 / 풀하우스.08 / **플러시.30 / 포카드.50 / 스트플.75**. **1차 밸런싱 완료**(전략 시뮬). 흔한 족보=소액(아슬아슬 보존) / 희소 족보=큰 가산(빌드 보상). 도구: `node tools/hand-frequency.cjs`(빈도) · `node tools/strategy-sim.cjs`(전략 비교).

## 5. 보스 블라인드 (현행 12종 — 코드 `BOSSES`/§6 = tmult SSoT, 아래는 대표 발췌)

| 보스 | 규칙 | tmult | 카운터 |
|---|---|---|---|
| ⤵ 내리막 (v3.26, seal_run 교체) | **오름차순** ±1 연결 무효(내림·같은무늬/숫자·와일드 정상) | 0.72 | 같은무늬/같은숫자, 내림 배치, 와일드 · 봉인=오름 체인 배율뿐(stair 부적 보너스 무영향, §6) |
| 🩸 단색의 저주 | 빨강(♥♦) 기본점수 0 | 1.0 | 검정(♠♣) 위주 · 안테1 가능 |
| ✋ 인색한 손 | 손패 1장 감소 | 0.58 | 손패확장 스킬 · act2 |
| 🗡 무딘 칼날 | 체인 배율 -1 | 0.85 | 더 긴 체인 · 안테1 가능 |
| 🔒 봉인된 무늬 | ♠ 카드 연결 불가 | 0.47 | 와일드, ♠ 덱에서 제거 · act3 |

> **안테1 보스 = act1 비-final 풀(단색저주·무딘칼날·보릿고개) 中 1**(전부 손패3 유지, 점수만 깎는 순한 보스). 도구 없는 첫 보스를 확정 워밍업으로. 손패 감소(인색한손)·연결규칙 봉인계는 act2+부터.

> **설계 의도**: 보스는 *숫자 벽*이 아니라 *규칙*으로 어렵게 → 플레이어가 **적응(덱 조정)**하게. 와일드 카드가 봉인계 보스의 명확한 카운터인 것은 의도된 깊이("이 보스엔 와일드를 사자").

### 🥇 포커 족보 보너스 — ✅ 구현됨(v3.4), 밸런싱 진행 중
> "체인을 길게" vs "플러시/포카드를 세팅"의 **두 전략축**. 발라트로식이되 우리 구조(8장 강제)에 맞춰 **텍사스 서열 라벨 + 빈도보정 가산**으로 구현(설계 WHY는 §3.2).

- ✅ **밸런싱 1차 완료** (`tools/strategy-sim.cjs`): ★핵심 발견 — **전략축은 "플레이 중 매 수 선택"이 아니라 "빌드 방향"**. 플러시 덱을 빌드해두면 그리디(점수최대)로 쳐도 플러시 38% 자연 달성 + 보너스(빌드덱 그리디 클리어 81→91%). "강제로 노리는" 플러시봇은 빌드해도 그리디보다 낮음(♥ 강제 = 체인 손해) → **빌드해두고 그리디로 치는 게 최적**(발라트로식, 매 수 부담 적음). 계수 .30/.50/.75로 상향 = 빌드 보상 확실 + 맨덱 강제는 손해 = 밸런스 안전.
